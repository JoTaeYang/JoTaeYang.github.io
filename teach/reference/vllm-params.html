vLLM 파라미터 치트시트

엔진(서버) 파라미터 = 시작 시 1회, 전원 공유. SamplingParams = 요청마다. 근거: docs.vllm.ai — Engine Arguments (stable, 2026-07 확인).

① 메모리 예산 (KV 캐시 예산을 정한다 — L2·L4)

플래그기본값역할 / 튜닝 방향
--gpu-memory-utilization0.92인스턴스가 쓸 GPU 메모리 비율. ↑ = KV 예산↑, 과하면 startup OOM.
--max-model-lenmodel config시퀀스 최대 길이(프롬프트+생성) = 시퀀스 1개의 KV 상한. 워크로드가 허용하면 줄여서 동시성 확보.
--kv-cache-dtypeautoKV 캐시만 양자화 (fp8, fp8_e5m2, fp8_e4m3). 캐시 용량 2배 + 디코드 읽기 바이트↓.
--block-size16PagedAttention 페이지 크기(토큰). 대개 건드리지 않음.
--swap-space4 (GiB)시퀀스별 CPU 스왑 공간(선점 시 사용).
--enable-prefix-cachingV1 기본 on프롬프트 공통 접두사의 KV 블록 재사용 — 시스템 프롬프트가 길고 공유될 때 prefill 대폭 절감.

② 스케줄러 (반복당 정원 — L4·L5)

플래그기본값역할 / 튜닝 방향
--max-num-seqscontext별 (~256)반복당 동시 시퀀스 상한. 실전에선 KV 예산이 먼저 바닥나는 경우가 많다.
--max-num-batched-tokenscontext별 파생반복당 토큰 예산 = chunked prefill 청크 크기. ↓ = ITL 안정, ↑ = prefill 처리량/TTFT 개선.
--enable-chunked-prefillV1 기본 on긴 prefill을 잘라 디코드와 한 배치에 섞기 (L5 그 자체).

③ 바이트 줄이기 / 읽기당 토큰 늘리기 (L6·L7·L9)

플래그기본값역할 / 튜닝 방향
--dtypeauto가중치·연산 기본 정밀도: bfloat16·float16·float32. 양자화 아님.
--quantization / -qNone가중치 양자화 방식: fp8·awq·gptq·bitsandbytes 등. 양자화된 체크포인트는 대개 자동 감지.
--speculative-configNoneJSON: {"method":"mtp","num_speculative_tokens":2}. method: mtp·eagle·eagle3·draft_model·ngram. num_speculative_tokens = K(L9).

④ 멀티 GPU

플래그기본값역할
--tensor-parallel-size1레이어 행렬을 N GPU에 세로 분할, 매 토큰 전원 참여. 노드 내(NVLink) 기본 수단.
--pipeline-parallel-size1레이어를 가로 분할(층별 담당). 주로 노드 간. 총 GPU = TP × PP.

⑤ 기타 자주 보이는 것

플래그역할
--enforce-eagerCUDA graph 비활성(디버깅용). 켜면 느려진다 — CUDA graph는 Phase 3/4 주제.
--served-model-nameAPI에 노출할 모델 이름 별칭.
--seed샘플링 재현성.
--max-logprobs요청당 반환 가능한 logprob 수 상한.

SamplingParams (요청 단위 — 성능이 아니라 출력 제어)

키역할
temperature분포 평탄화(0 = greedy). L9의 p·q 분포가 이것의 영향을 받는다.
top_p / top_k누적확률 / 상위 k개로 후보 절단.
max_tokens생성 길이 상한 — 분포가 길수록 서버 KV 예산을 오래 점유.
stop / stop_token_ids중단 문자열/토큰.
n요청당 후보 수 (KV는 n배 아님 — 프리픽스 공유).
