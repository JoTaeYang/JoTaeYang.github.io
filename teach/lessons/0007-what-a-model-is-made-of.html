Lesson 07 · Phase 1–2 · Foundations (개념의 바닥)

모델은 결국 숫자다: 파라미터·레이어·임베딩

"70B 파라미터"가 실제로 무엇이고, 무슨 일을 하며, 왜 매 토큰 전부 읽어야 하는가 — 곁가지 질문들이 모여 만든, 코스의 밑바닥.

이번 레슨의 단 하나의 승리 "모델 = 학습된 숫자 덩어리"라는 한 문장을 완전히 분해한다: 파라미터당 바이트 → 파라미터의 역할 → 레이어 → 임베딩 벡터 → 벡터 값의 의미. 이걸로 L1(왜 전체를 읽나)과 L6(양자화)이 바닥까지 닫힌다.

1"파라미터당 바이트"란

모델은 엄청나게 많은 숫자 덩어리다. 그 숫자 하나하나가 파라미터(=가중치). "70B 모델" = 700억 개의 숫자. 각 숫자는 학습으로 정해진 실수(예: 0.7314…)다.

컴퓨터는 숫자를 비트(0/1)로 저장하는데, 숫자 하나에 비트를 몇 개 쓸지는 선택이다. C의 타입을 떠올리면 정확하다:

int   x;  // 4바이트
short y;  // 2바이트
char  z;  // 1바이트

실수도 같다. 같은 숫자 하나를 어떤 포맷에 담느냐에 따라 차지하는 바이트가 다르다 — 이게 파라미터당 바이트:

포맷비트파라미터당 바이트같은 값 0.7314 저장 시
FP16162.0 B0.7314 (거의 그대로)
FP881.0 B0.73 (반올림)
FP440.5 B (둘이 1바이트)0.75 (거칠게)

사진을 고/저해상도로 저장하는 것과 같다 — 같은 내용, 파일 크기만 다르고, 저해상도는 디테일이 날아간다(정확도 손실). 그래서 모델 총 바이트 = 파라미터 수 × 파라미터당 바이트. 양자화는 뒤 항만 줄인다(개수는 그대로).

2700억 개의 숫자가 하는 일

신경망의 기본 동작은 단순하다 — 입력 숫자에 가중치를 곱해 더하기:

출력 = w1·x1 + w2·x2 + w3·x3 + … + b ↑ 이 w(weight)들이 파라미터

LLM에서 한 레이어가 하는 일은, 토큰 벡터를 거대한 가중치 행렬로 곱해 새 벡터로 바꾸는 것:

[8,192개 숫자] × [8,192 × 8,192 가중치 행렬] → [8,192개 숫자] 입력 토큰 ← 이 행렬이 파라미터 6,700만 개 → 변환된 토큰

행렬 하나가 벌써 8,192² ≈ 6,700만 개. 한 레이어엔 이런 행렬이 여러 개(어텐션의 Q·K·V 만드는 행렬 + FFN 행렬), 그런 레이어가 ~80층 → 합쳐서 수백억 개 = "70B". 파라미터는 거대한 믹싱 콘솔의 700억 개 노브이고, 학습이 그 값을 정하고, 추론은 고정된 노브로 계산만 흘려보내는 것이다.

3레이어란 무엇인가

레이어 = 토큰 벡터들을 한 번 다듬어 내보내는 처리 단계 하나. 입력과 출력의 모양이 같아서 똑같은 구조를 계속 이어 붙일 수 있다. 시스템 관점으론 함수 합성:

출력 = f₈₀( … f₂( f₁( 입력 ) ) ) 각 fᵢ: 구조는 같고(벡터열→벡터열), 내부 상수(파라미터)만 다름 = 파이프라인 스테이지

레이어 하나 안에는 두 하위 단계가 있다:

하위 단계하는 일비유
① 어텐션 (L3)토큰들이 서로 정보 교환 ("그것"↔"동물")회의: 서로 대화
② FFN (피드포워드)각 토큰이 혼자서 그 정보를 곱씹어 변환회의 후 각자 정리

(residual 연결·정규화도 끼지만 지금은 "어텐션 + FFN 한 세트"로 충분.) 층을 지날수록 벡터는 더 추상적 의미를 담는다 — 앞쪽 층: 문법·표면, 뒤쪽 층: 문맥·의도. 한 번으론 부족하니 80번 정제한다.

4토큰은 왜 벡터로 표현되나

신경망은 숫자만 곱할 줄 안다. 글자 "동물"은 곱할 수 없으니 숫자로 바꿔야 한다. 그런데 숫자 하나면 모든 단어가 일직선 위 한 점이라 다면적 관계를 못 담는다. 그래서 벡터(숫자 여러 개) = 의미 공간의 좌표를 쓴다:

"동물" = [ 0.8, 0.1, 0.9, -0.3, … ] ← 8,192개 "고양이" = [ 0.7, 0.2, 0.9, -0.2, … ] ← "동물"과 비슷한 방향 "길" = [ -0.4, 0.9, -0.1, 0.6, … ] ← 전혀 다른 방향

비슷한 의미 → 가까운 벡터. 그래서 king − man + woman ≈ queen 같은 벡터 산술이 성립한다. 그리고 이것이 L3의 "내적 = 유사도"를 닫는다: 두 벡터가 같은 방향이면 내적이 크다 = 비슷한 의미 = 높은 관련도. 토큰을 벡터로 만들었기에 어텐션이 내적으로 관련도를 잴 수 있다.

토큰→벡터 변환은 임베딩 테이블(조회표)로 하며, 이 표의 숫자들도 학습된 파라미터다.

5벡터 값(+1/−1)의 의미와, 축은 어떻게 의미를 갖나

값 = 각 축에서의 좌표. 부호는 방향, 크기는 강도, 그리고 상대적이다:

어떤 축이 (느슨하게) "생물 ↔ 무생물"이라면: "동물" → +0.9 (생물 쪽으로 강하게) "돌" → -0.9 (무생물 쪽으로 강하게) "그것" → 0.1 (이 축에선 중립/무관)

값이 대략 −1~1에 머무는 건 우연이 아니라 정규화(normalization) 층이 벡터 크기를 눌러 학습을 안정시키기 때문.

축의 의미는 아무도 안 정한다 — 학습으로 창발한다 사람이 "1번 축 = 생물다움"이라 지정하지 않는다. 처음엔 전부 랜덤 → 목표는 오직 "다음 토큰 맞히기" → 틀릴 때마다 경사하강법이 700억 노브를 아주 조금씩 "예측이 나아지는 방향"으로 돌림 → 수조 토큰 반복 → 잘 맞히기에 유리한 배치로 공간이 스스로 정렬된다.
정직한 교정: 축 하나 = 개념 하나가 아니다 위에서 "1번 축=생물다움"이라 한 건 이해를 돕는 단순화다. 실제 의미는 개별 축이 아니라 여러 축의 조합(공간 속 '방향')에 분산되어 있다 — 분산 표현(distributed representation). 그래서 숫자 하나만 떼면 대개 해석 불가. "성별" 방향이 공간 어딘가 학습으로 생겼기에 king−man+woman≈queen이 되는 것.

6다 합치면: "그것"이 "동물"인 걸 어떻게 알아내나

지금까지의 조각(임베딩 벡터 + 어텐션 + 레이어)이 실제로 어떻게 대명사를 푸는지 한 문장으로 따라가 보자:

"그 동물은 길을 건너지 않았다, 너무 피곤했기 때문에. 그것은 쉬고 싶었다."

핵심 전제: 임베딩 직후 "그것"의 벡터엔 "동물"을 가리킨다는 정보가 없다. 그 연결은 레이어를 지나며 어텐션이 문맥을 섞어 넣을 때 생긴다.

  1. "그것"이 Query를 만든다 (학습된 W_Q로). 뜻: "앞에 나온, 지치고 쉬고 싶어 할 만한 개체(명사)를 찾아라."
  2. 과거 토큰이 Key를 만든다 (W_K로): K("동물") ≈ "생물 명사, 지칠 수 있음" / K("길") ≈ "무생물, 장소".
  3. Q·K 내적으로 점수 (§5의 '내적=유사도'):
    Q("그것") · K("동물") → 방향 정렬 → softmax 0.70 Q("그것") · K("길") → 방향 어긋남 → softmax 0.08
    L3에서 봤던 그 0.70/0.08이 여기서 나온다.
  4. Value 가중합 → 흡수: 큰 가중치로 V("동물")이 "그것"의 새 벡터에 섞여, "그것" 벡터 안에 "동물"의 의미가 실제로 담긴다.
왜 "길"이 아니라 "동물"인가 — 의미 단서 단순히 "가장 가까운 명사"를 고르는 게 아니다. "피곤했기 때문에", "쉬고 싶었다"는 생물에게나 해당한다. 그래서 "그것"의 Query가 찾는 방향(지칠 수 있는 개체)이 K("동물")과 정렬되고 K("길")과는 어긋난다. 이 판단은 §5의 학습으로 생긴 방향들(생물다움·피곤함)로 이뤄진다 — 그 유명한 Winograd 스키마 추론.

이 정렬이 가능한 건 경사하강법(§5)이, 지시 대상을 맞게 이었을 때 예측이 좋아지는 걸 보고 W_Q·W_K를 "대명사 Query가 알맞은 선행 명사 Key와 정렬되도록" 수조 문장에 걸쳐 다듬었기 때문이다. 그리고 이건 한 방에 되는 게 아니라 여러 헤드가 서로 다른 가설(가까운 명사·주어·생물 개체)을 검사하고, 여러 층을 거치며 확정된다.

정직한 한계: "이 뉴런이 코어퍼런스 담당"이라 딱 짚진 못한다 — 판단은 §5처럼 분산·창발적이다. 하지만 메커니즘(대명사 Q ↔ 알맞은 명사 K 정렬 → V 흡수, 학습으로 조정, 층·헤드에 걸쳐)은 위 그대로다.

전체를 하나로

토큰 "동물" ↓ 임베딩 테이블(파라미터) [벡터] ← 의미 좌표 (부호=방향, 크기=강도) ↓ 레이어 80개 통과 (매 층: 어텐션+FFN, 자기 파라미터로 곱셈) [정제된 벡터] ↓ 다음 토큰 예측 ⇒ 매 토큰: 700억 파라미터를 전부 통과 = 가중치 전체 읽기 (L1) ⇒ 각 숫자를 몇 바이트로 저장? = 양자화 (L6)
임베딩 벡터에서 각 차원(축)의 '의미'는 어떻게 정해지나?
스스로 확인 (retrieval practice) 종이를 덮고: ⑴ 양자화가 줄이는 건 파라미터 '수'인가 '파라미터당 바이트'인가? ⑵ 레이어를 함수로 비유하면? ⑶ 토큰을 '숫자 하나' 아닌 '벡터'로 두는 이유는? ⑷ 축의 의미는 누가 정하나? ⑸ "그것→동물" 연결은 어느 단계에서, 무엇의 정렬로 생기나?
