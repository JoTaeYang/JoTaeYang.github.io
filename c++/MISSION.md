# Mission

## Why

이 저장소의 C++ IOCP 게임서버(NetLib + FighterServer)를 **아무것도 안 보고 손코딩으로 처음부터 다시 작성할 수 있는 수준**의 실무 역량을 갖춘다. 취업·면접이 아니라 엔지니어링 실력 자체가 목적이다.

## What "done" looks like

- 빈 프로젝트에서 IOCP 기반 네트워크 라이브러리(링버퍼, 세션 관리, 패킷 직렬화, 메모리 풀, 락프리 구조)를 스스로 설계하고 구현할 수 있다.
- 각 설계 결정(왜 링버퍼인가, 왜 CAS인가, 왜 IOCP인가)을 대안과 비교하며 설명할 수 있다.
- 밑받침이 되는 CS 기초 — C++ 언어/메모리 모델, OS(스레드, 가상 메모리, 커널/유저 모드), 네트워크(TCP/IP, 소켓), 자료구조 — 를 코드와 연결해서 이해한다.

## Current state (2026-07-06)

- 코드를 따라 짜본 경험은 있으나 **C++ 자체가 아직 불안** (포인터, 메모리, 클래스 등 기본기부터 다지고 싶음).
- 기존 코드베이스: NetLib(IOCP 서버 라이브러리) + FighterServer(게임 로직) + Go 포트 + 벤치마크가 이미 존재 — 학습의 살아있는 교재로 사용한다.

## Constraints & preferences

- 기한 없음. **깊이 우선** — 시간이 걸려도 각 주제를 바닥까지 판다.
- (2026-07-06 확정) **최단 경로가 아니라 완전판을 간다**: 전체 지도(reference/full-curriculum.html)의 기둥 A~F 전 항목을 순회한다. 심화 레슨(A6~A13, B6~B11, C6~C9)도 건너뛰지 않는다. 약 65개 레슨 + 재구현 프로젝트 규모를 받아들임.
- 기본 CS 지식도 커리큘럼에 반드시 포함.
- 언어: 한국어로 학습.

## Curriculum pillars

1. **C++ 언어 기본기** — 메모리 모델, 포인터/참조, 클래스와 객체 수명, RAII, 템플릿
2. **CS 기초** — 자료구조, 프로세스/스레드, 가상 메모리, 캐시, 커널/유저 모드
3. **네트워크** — TCP/IP, 소켓 API, 블로킹/논블로킹, 패킷 설계와 직렬화
4. **시스템 프로그래밍 (Windows)** — WinAPI, IOCP, 비동기 I/O, 크래시 덤프
5. **멀티스레드** — 동기화 프리미티브, 원자 연산과 CAS, 락프리 자료구조, 메모리 오더링
6. **종합** — NetLib/FighterServer를 처음부터 재구현
