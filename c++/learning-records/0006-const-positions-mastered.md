# 파생 4-1 — const 위치 문법, 첫 시도 전부 정답

2026-07-12, 레슨 4 진행 중 사용자가 스스로 질문 제기: "const 위치에 따라 의미가 달라지는 문법은 알 필요 없나?" → 파생 문서 [[0004-1-const-positions]] 즉석 제작(실제 코드 앵커: `const WCHAR* cwIP`, `const vector<CCharacter*>& GetSector`). 드릴 3 + 퀴즈 2 **전부 정답**.

- `*` 기준 좌/우 const 규칙(대상 불변 vs 포인터 불변) 적용 성공.
- "const는 한 겹만 막는다"(const vector& 는 벡터를 잠그지만 원소 CCharacter*가 가리키는 캐릭터는 수정 가능) 함정 문제도 정답 — 규칙을 새 상황에 전이함.
- "const 유무 = 입력이냐 출력이냐"(DomainToIP의 `const WCHAR*` 입력 vs `IN_ADDR*` 출력) 드릴도 정답.

## Implications

- **패턴 확인**: 사용자는 레슨이 당연시하고 넘어간 전제를 스스로 감지해 질문함(사전할당 이유 정정에 이어 두 번째). 이 "능동적 반박·질문" 성향이 강점 → NOTES의 방침(레슨을 반박 가능한 형태로) 유지.
- const-correctness는 레슨 17(Rule of Three: 복사 금지), 레슨 4 본문(const T&)에서 재사용 → 별도 재교정 불필요, 자연 강화될 것.
- 갱신(같은 날): **레슨 4 본문 퀴즈도 전부 정답** → 값/포인터/참조 전달 방식 + 출력 인자·nullptr 판단까지 통과. **레슨 5(상속·virtual) 진입 승인 (2026-07-12)** — 배치 1의 마지막 편.
