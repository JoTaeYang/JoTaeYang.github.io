## 이 세션에서 드러난 것

- **상승 곡선이 뚜렷함**: A3까지는 오답이 있었으나 A4·A5·4-1은 첫 시도 전부 정답. C++ 기본기 자체가 실제로 올라옴("C++이 불안하다"던 [[0001]] 시점의 자가진단은 이제 낡음).
- **사용자 주도 질문 2회**가 파생 문서를 낳음(사전할당 이유 정정 [[0004]], const 위치 [[0006]], 힙 비용). 레슨이 당연시한 전제를 감지하는 능력이 강점.
- **개념 연결이 일어남**: 파생 3-1(할당자가 헤더를 쓴다)이 A3 퀴즈 2 오답(delete/delete[] UB)의 *메커니즘적 이유*를 사후에 설명해줌 — 암기가 이해로 전환된 사례.

## 남은 격차 (의도적으로 수용됨)

- **재인 vs 생성**: 지금까지 전부 객관식·설명 기반. 백지에서 코드를 쓴 적은 **0회**. 미션의 최종 목표가 손코딩임에도 그렇다. 에이전트가 레슨별 손코딩 과제를 제안했으나 **사용자가 "기둥 F에서 몰아서"를 선택**(2026-07-12). 결정 존중.
- **귀결**: 그 격차를 막을 유일한 장치가 간격 복습이 됨. 오늘 6일 만에 중간 패딩 규칙이 흐려진 사례([[0002]] 갱신)가 이 위험의 실증. → 앞으로 **퀴즈 통과만으로 넘기지 않고, 보기 없는 자유회상 재검을 더 자주** 건다(NOTES 반영).
- **면접은 목표 아님 재확인**: 사용자가 "면접에 먹힐까" 물었으나 미션 불변 확정. MISSION.md 수정 없음.

## 다음 간격 재검 후보 (며칠 뒤, 보기 없이 말로)

1. `delete` vs `delete[]`를 섞으면 왜 UB인가 — **할당자 헤더**까지 언급하며 설명할 수 있는가
2. 스택 할당이 힙보다 싼 이유를 명령어 수준(`sub rsp`)과 절차(락·검색·헤더·시스템 콜)로 대비
3. `const WCHAR*` vs `WCHAR* const` — `*` 좌우 규칙
4. virtual을 빼면 정확히 무엇이 깨지는가
