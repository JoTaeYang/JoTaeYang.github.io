사용하는 Windows API

CreateIoCompletionPort, GetQueuedCompletionStatus, PostQueuedCompletionStatus, overlapped WSARecv/WSASend, _beginthreadex, timeBeginPeriod(1), InterlockedCompareExchange128, MiniDumpWriteDump. 뮤텍스/SRWLock 없음 — 의도적으로 전부 락프리.

패킷 생애주기 (외워야 할 흐름)

  1. WSARecv 완료 → 워커 스레드가 RecvMsgDiv에서 링버퍼로부터 패킷 재조립, 코드(0x89)·길이(≤4096) 검증
  2. 완성된 패킷을 completeRecvQ에 넣고 NotifyRecv
  3. GameThread가 5ms 틱마다 드레인(세션당 ≤2000개) → OnRecv → PacketProc_* 핸들러가 게임 상태 변경 + 섹터 브로드캐스트
  4. SendPacket: AddRef + SendQ 인큐 + NotifySend
  5. SendThread가 2ms 틱마다 최대 100버퍼를 하나의 WSASend로 배칭 → 완료 시 SendUpdate가 DecRef

게임 프로토콜 (Protocol.h)

4바이트 헤더 { BYTE byCode=0x89; WORD wSize; BYTE byType; } (v2에서 길이를 BYTE→WORD로 확장, 최대 페이로드 4096). 8방향 이동, 33×43 섹터 기반 관심영역 관리. 캐릭터 생성/삭제/이동/공격1-3/데미지/SYNC/에코.

가르칠 가치가 있는 기법 목록

  • 락 없는 세션 수명 관리: lIOCount + lSendFlag(0/1/2) + 세션 Key = 17비트 인덱스 + 47비트 unique
  • 좀비 커넥션 타임아웃 (30초 idle 스윕)
  • 프로토콜 방어: 코드/길이 검증, 직렬화 경계 검사 → 즉시 연결 해제
  • DCAS + unique 태그로 ABA 문제 방지
  • 섹터를 std::list→std::vector로 바꾼 캐시 지역성 개선 + swap-remove
