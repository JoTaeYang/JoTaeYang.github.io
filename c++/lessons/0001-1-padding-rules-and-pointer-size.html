Lesson 0001-1 · 파생 심화 · 레슨 1에서 파생

패딩의 두 규칙, 그리고 포인터의 크기

레슨 1 퀴즈에서 드러난 두 혼동을 정면으로 교정한다. 짧지만, 이 둘이 잡혀야 레슨 2(포인터)가 제대로 선다.

혼동 ① — 패딩에는 규칙이 두 개다 (섞으면 틀린다)

규칙 A: 중간 패딩 — 범인은 "바로 다음 멤버"

멤버 사이의 패딩은 오직 하나의 질문으로 결정된다: "바로 다음 멤버가 요구하는 정렬은 몇인가?" 구조체 전체 크기(8의 배수)와는 아무 상관이 없다.

short shY;                  // 오프셋 16–17. 다음 위치는 18.
st_SECTOR_POS CurSectorPos; // 첫 멤버가 int → 4정렬 요구.
                            // 18은 4의 배수가 아님 → 20까지 2바이트 패딩.

만약 shY 다음이 char였다면? 패딩 0 — char는 1정렬이라 18에 바로 앉는다. 다음이 __int64였다면? 24까지 6바이트 패딩. 패딩의 양은 전적으로 다음 손님이 정한다.

규칙 B: 꼬리 패딩 — 범인은 "배열의 다음 원소"

구조체 끝의 패딩만이 "전체 크기 = 최대 멤버 정렬의 배수" 규칙을 따른다. 이유는 배열이다: arr[1]이 arr[0] 바로 뒤에 붙을 때 arr[1]의 8정렬 멤버(SessionKey)가 어긋나면 안 되므로, 41을 48로 올린다.

중간 패딩 (규칙 A)꼬리 패딩 (규칙 B)
위치멤버와 멤버 사이마지막 멤버 뒤
기준바로 다음 멤버의 정렬최대 멤버 정렬의 배수로 올림
누굴 위해다음 멤버 하나배열의 다음 원소 전체

드릴 — 손으로 세 개만 (답은 아래 접기)

struct P1 { char a; int b; };                 // sizeof?
struct P2 { int b; char a; };                 // sizeof?  (P1과 멤버 같음, 순서만 반대)
struct P3 { char a; __int64 b; short c; };    // sizeof?
답 확인 (계산을 먼저 끝내고 열 것)

P1 = 8: a(0), 패딩 3(규칙 A: int가 4정렬), b(4–7). 끝이 8 — 4의 배수라 꼬리 패딩 0.
P2 = 8: b(0–3), a(4). 5를 4의 배수로 올림(규칙 B) → 8. 같은 멤버인데 패딩의 위치가 다르다!
P3 = 24: a(0), 패딩 7(규칙 A: __int64), b(8–15), c(16–17), 꼬리 패딩 6(규칙 B: 18→24). 순서를 b,c,a로 바꾸면 16이 된다 — 레슨 1 떡밥의 원리다.

혼동 ② — 포인터의 크기는 대상과 무관하다

명함과 집

포인터가 담는 것은 객체가 아니라 주소 하나다. 집(CCharacter, 48바이트)이 아무리 커도 명함에 적힌 주소(x64에서 8바이트) 크기는 같다. 초가집(char, 1바이트)의 주소도 같은 8바이트다.

sizeof(char)        == 1        sizeof(char*)        == 8
sizeof(CCharacter)  == 48       sizeof(CCharacter*)  == 8
sizeof(double[100]) == 800      sizeof(double(*)[100]) == 8   // 배열 포인터마저도!

왜 8인가? x64 CPU의 주소가 64비트(8바이트)이기 때문 — 가리키는 대상이 무엇이든 위치를 표현하는 데 드는 비용은 같다. 그럼 타입(CCharacter* vs char*)은 왜 붙이나? 크기 때문이 아니라 해석 때문이다: 역참조할 때 몇 바이트를 어떤 구조로 읽을지, 그리고 +1 할 때 몇 바이트 건너뛸지(48 vs 1)를 컴파일러가 알아야 하니까.

한 문장 정리
포인터의 크기는 플랫폼이 정하고(x64 = 8), 포인터의 타입은 해석과 보폭을 정한다. 이 문장이 외워지면 레슨 2의 players + index 연산이 자명해진다.

30초 검증 (직접 실행)

#include <cstdio>
#include <windows.h>
#include "../FighterServer/CCharacter.h"

int main() {
    printf("%zu %zu %zu %zu\n",
        sizeof(char*), sizeof(int*), sizeof(CCharacter*), sizeof(void*));
    // 전부 8이 나온다. 하나라도 다르면 스크린샷 들고 에이전트에게 올 것.
}
