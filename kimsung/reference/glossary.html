용어집

이 코스에서 쓰는 말들. 레슨마다 여기 정의를 따릅니다. 한글 옆의 영어는 실무에서 만날 표기입니다.

기본

머신러닝 Machine Learning · ML

사람이 규칙을 일일이 짜주는 대신, 데이터를 보고 프로그램이 스스로 규칙을 배우는 방식. 1959년 아서 새뮤얼(Arthur Samuel)이 제안.

예: 스팸 메일을 "이런 단어 있으면 스팸"이라고 일일이 정하는 대신, 스팸/정상 메일을 잔뜩 보여주고 스스로 판단하게 함.

명시적 프로그래밍 Explicit Programming

개발자가 "이 경우엔 이렇게, 저 경우엔 저렇게"를 전부 직접 정해 놓는 전통적 방식. 규칙이 너무 많은 문제(스팸, 자율주행)에선 한계.

모델 Model

데이터로 학습한 결과물. 새 입력을 주면 답(예측)을 내놓는 "학습된 프로그램".

학습의 종류

지도 학습 Supervised Learning

정답표(레이블)가 달린 데이터로 배우는 방식. "감독관(supervisor)"이 정답을 알려주며 가르치는 그림. 이 코스에서 주로 다루는 것.

예: 고양이 사진마다 "cat" 딱지를 붙여서 학습 → 새 사진이 고양이인지 맞힘.

비지도 학습 Unsupervised Learning

정답 레이블 없이, 데이터 안의 비슷한 것끼리 스스로 묶는 방식.

예: 구글 뉴스가 비슷한 기사끼리 자동으로 그룹핑.

레이블 Label

데이터에 붙은 정답. 지도 학습에서 보통 y로 표기.

특징 / 피처 Feature

예측에 쓰는 입력값. 보통 x로 표기. (예: "공부한 시간")

트레이닝 데이터셋 Training Data Set

모델을 학습시키는 데 쓰는, 입력(x)과 정답(y)이 짝지어진 데이터 모음.

테스트 값 x_test

학습이 끝난 모델에게 "이건 답이 뭐야?"라고 물어보는, 정답을 모르는 새 입력.

지도 학습의 세 갈래

회귀 Regression

답이 넓은 범위의 숫자인 문제. "얼마나?"를 맞힘.

예: 공부 시간 → 시험 점수(0~100점) 예측.

분류 (이진 분류) Classification · Binary

답이 두 개 중 하나인 문제. "이거야 저거야?"를 맞힘.

예: 합격(pass) vs 불합격(non-pass), 스팸 vs 정상(ham).

다중 분류 Multi-label / Multi-class Classification

답이 여러 개 중 하나인 문제.

예: 학점 A·B·C·D·F 중 하나 예측.

선형 회귀

선형 회귀 Linear Regression

데이터에 가장 잘 맞는 직선 하나를 찾는 회귀 방법. "입력이 커지면 출력도 일정하게 커진다"는 관계를 직선으로 모델링. 실무에서 가장 기본이자 널리 쓰임.

가설 Hypothesis · H(x)

데이터에 맞을 거라고 세우는 선의 식: H(x) = Wx + b. H(x)는 이 선이 내놓는 예측값.

가중치 (기울기) W · Weight

선이 얼마나 가파른가. x가 1 늘 때 예측 y가 얼마나 오르는지.

편향 (y절편) b · Bias

선이 세로로 어디 걸려 있나. x가 0일 때의 예측값.

비용 함수 Cost / Loss Function

선이 얼마나 틀렸는가를 하나의 숫자로. 각 점의 오차 (H(x) − y)를 제곱해서 평균낸 값. 작을수록 좋은 선. loss라고도 부름.

제곱하는 이유: ① +/− 상쇄 방지, ② 크게 틀린 점에 더 큰 벌점.

최소화 Minimize

선형 회귀 학습의 목표: cost(W, b)를 가장 작게 만드는 W와 b를 찾는 것. 그 방법이 경사하강법.

경사하강법

경사하강법 Gradient Descent

cost를 최소화하는 W(·b)를 기계적으로 찾는 방법. 아무 값에서 시작해, 지금 지점의 경사(기울기)를 재고 내리막으로 한 발짝씩 옮기길 반복 → 밥그릇 바닥(cost 최소)에 도달.

비유: 안대 쓰고 언덕에서, 가장 가파른 내리막으로 한 발짝씩 내려오기.

기울기 / 미분 Gradient · Derivative

한 지점에서 곡선이 얼마나 기울었나를 재는 자. 경사하강법에서 내리막 방향을 정함. 손으로 계산할 일 없음 — 컴퓨터가 대신함.

학습률 Learning Rate · α

한 발짝의 보폭. 보통 0.01처럼 작은 값. 너무 크면 바닥을 지나쳐 튕기고, 너무 작으면 느림.

볼록 함수 Convex Function

웅덩이가 하나뿐인 매끈한 밥그릇 모양. cost가 볼록이면 경사하강법이 시작점과 무관하게 항상 최저점을 찾는다고 보장. 선형 회귀의 cost가 이 모양.

여러 입력 (multi-variable)

다변수 선형 회귀 Multi-variable Linear Regression

입력(피처)이 여럿인 선형 회귀. 피처마다 weight를 하나씩 붙임: H = w₁x₁ + w₂x₂ + w₃x₃ + b. cost·경사하강법은 입력 1개일 때와 그대로.

예: 퀴즈1·퀴즈2·중간 점수(피처 3개) → 기말 점수 예측.

행렬 · 행렬 곱 Matrix · Matrix Multiplication · XW

숫자를 네모나게 늘어놓은 표. 긴 가설을 한 덩어리로 접는 도구. 가설을 H(X) = XW로 씀(구현 관례로 X를 앞에; 값은 Wx와 동일).

인스턴스 Instance

데이터 한 줄(한 샘플). 학생 5명이면 인스턴스 5개. X를 아래로 쌓으면 여러 인스턴스를 한 번의 행렬 곱으로 계산.

셰이프 (크기) Shape · [행, 열]

행렬의 크기 [행, 열]. 곱셈은 안쪽 값이 맞아야 되고 바깥 값이 결과: [5,3]×[3,1]=[5,1]. W = [피처 수, 출력 수]. 인스턴스 자리는 가변이라 None(TF) 또는 -1(넘파이).
