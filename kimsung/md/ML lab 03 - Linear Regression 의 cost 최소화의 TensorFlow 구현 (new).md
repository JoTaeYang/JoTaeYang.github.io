## 코스트 함수 다시 보기

우리가 직접 텐서플로를 사용하기 전에, 이론 시간에 다루었던 그 하이포시시스(hypothesis)를 한 번만 더 보고 들어갈 텐데요, 여기서는 좀 간략하게 된 형태를 사용하겠습니다.

우리가 관심 있는 게 이제 코스트 함수를 보는 건데요, 그러기 위해서 좀 쉽게 보기 위해서 우리 hypothesis를 `W * x`로 둡니다. 원래는 `W * x + b`이고 바이어스 텀(bias term)을 쓰지만, 여기서는 이 부분은 생략하도록 하겠습니다.

$$ H(x) = Wx $$

그런 다음에 우리가 이 코스트 함수는 마찬가지로 `Wx`와 `y` 차의 제곱의 평균이 되겠죠. 자, 이렇게 나타나는데 이것을 우리가 코스트(cost)라고 두었고, 이렇게 보니까 이제 코스트는 `W`의 함수가 됩니다.

$$ cost(W) = \frac{1}{m} \sum_{i=1}^{m} (Wx_i - y_i)^2 $$

그 이야기는 `W` 값이 바뀔수록 코스트가 바뀌게 되고, 미니마이즈 한다는 것은, 이 코스트를 미니마이즈 하자라는 것은, 이 코스트가 최저가 되는 이 `W` 값을 찾자는 이야기가 되겠죠.

## 코스트 함수 시각화하기

자, 그러면 우선 우리가 처음에 해 보고 싶은 것은 이 코스트라는 것이 어떻게 생겼을까 한번 보려고 하는데요. 그것을 위해서 우리가 텐서플로를 이용할 거고요. 그래서 이렇게 텐서플로를 임포트(import) 하고요, 그리고 우리가 굉장히 또 많이 사용하는 매트플롯립(Matplotlib) 라이브러리를 사용할 텐데요, 이것을 위해서 설치를 안 하셨다면 하시면 좋고요. 역시 이것도 이름이 기니까 `plt`라는 이름으로 사용하게 될 겁니다.

```python
import tensorflow as tf
import matplotlib.pyplot as plt
```

그런 다음에 우리가 가지고 있는 데이터, 이전 실습 비디오와 똑같이 간단하게 1, 2, 3이 주어지면 `y`도 1, 2, 3이 되는 굉장히 단순한 형태로 우리가 X, Y 데이터를 주고요.

```python
X = [1, 2, 3]
Y = [1, 2, 3]
```

우리가 이제 `W`에 해당되는 하이퍼파라미터를 `H(x) = x * W` 이렇게 간략한 것처럼 이렇게 줄 텐데요. `X`가 주어졌고 `W`에 해당되는 것을 우리가 이런 형태로 플레이스홀더(placeholder)로 줍니다. 그 이야기는, 우리가 이 값을 한번 바꿔가면서 그 값이 어떻게 되는지 보겠다는 거고요.

```python
W = tf.placeholder(tf.float32)
hypothesis = X * W
```

그런 다음에 코스트는 똑같이 hypothesis와 `Y`의 차를 이렇게 스퀘어(square)한 다음에, 즉 제곱한 다음에 그것을 평균 낸 형태입니다. 자, 이렇게 우리가 그래프를 구성했고요.

```python
cost = tf.reduce_mean(tf.square(hypothesis - Y))
```

자, 그런 다음에 이제 그래프를 구동하기 위해서 세션을 열고, 이 베리어블(variable)을 초기화시키고, 이제 그래프를 구동시킬 준비를 합니다.

```python
sess = tf.Session()
sess.run(tf.global_variables_initializer())
```

이때 우리가 그래프를 그리면서 `W`의 값과 그리고 코스트의 값을 저장할 리스트를 만들고요, 그래프를 그리기 위한 거죠.

```python
W_val = []
cost_val = []
```

그런 다음에 이 레인지(range)를 약간 -30에서 50 정도까지 이동시키면서, 이를 우리가 곱하기 0.1 하니까 대략 -3에서 5까지 움직이겠다, 그리고 0.1 간격으로 작게 움직이겠다는 겁니다. 자, 그렇게 하면서 이를 이렇게 피드(feed) 하죠. 그래서 그렇게 `W` 값을 넘기면서 이 코스트와 `W`가 어떻게 변하는지를 우리가 여기다가 값에 넣고 이것을 리스트에 넣게 됩니다.

```python
for i in range(-30, 50):
    feed_W = i * 0.1
    curr_cost, curr_W = sess.run([cost, W], feed_dict={W: feed_W})
    W_val.append(curr_W)
    cost_val.append(curr_cost)
```

자, 그렇게 한 다음에 우리가 하고 싶은 건 뭘까요? 이 `plt`를 이용해서 이 `W` 밸류가 X축이고 Y축이 코스트가 되는 어떤 그래프를 한번 그려 보자는 거죠. 그래서 실제로 우리가 이 코스트라는 함수가 어떤 모양을 하고 있는지 한번 시각화를 해보자는 것입니다.

```python
plt.plot(W_val, cost_val)
plt.show()
```

## 코스트 함수의 모양과 경사하강법

자, 여러분들이 예상한 것처럼 우리 코스트 함수는 이런 형태가 됩니다. 이쪽 X축이 `W`가 될 거고요, 이 축이 바로 코스트가 될 겁니다. 여기에서 이 코스트가 이런 밥그릇 모양(bowl 모양)으로 생겼다고 했죠. 우리가 코스트 펑션(cost function)이라고 부르는데, 이 경사면 따라 내려가기 알고리즘, 즉 경사하강법(Gradient Descent)을 적용하기에 굉장히 좋은 형태의 모양을 가지고 있습니다.

자, 그래서 우리가 이제 그림을 보면 알겠지만 대략 이 값을 최소화하는 값은 `W = 1`인 것이죠. 자, `W`가 1이면 우리가 이 코스트를 최적화하는데, 이 값을 자동으로 찾는 것입니다. 그래서 우리가 그것을 한번 해보게 될 텐데요.

한번 이 부분을 자세하게 보면, 우리가 이론 시간에 이야기한 것처럼 미분을 할 겁니다. 미분 값을 사용하는데요, 그 이유는 미분이란 것이 어떤 한 점에서의 기울기를 나타내는 거죠. 이 점에서 기울기는 이렇게 되겠고, 그래서 그 기울기를 보고, 예를 들어서 이런 경우에는 기울기가 뭘까요? 부호만 본다면, 우리가 기울기를 수학 기호로 이렇게 표시하는데, 이 기울기가 뭐가 되냐면 플러스가 되겠죠.

이렇게 그래프가 이렇게 돼 있으면 그래프는 플러스 그래프, 이게 이렇게 되면 이제 마이너스 그래프가 되겠죠. 자, 이렇게 되면 이게 이제 기울기가 플러스가 되죠. 그러면 우리는 어떻게 해야 될까요? `W`는 이 안쪽으로 가야 되겠죠. 우리가 여기에 더 있으니 안쪽으로 가야 되니까, 이 안쪽으로 이렇게 움직여, 이 `W`는 마이너스 방향으로 움직여야 됩니다.

자, 이렇게 볼 수가 있겠고 만약에 이쪽에 있으면 어떻게 될까요? 이쪽에 있으면 기울기가 이렇게 되니까 이 기울기는 마이너스가 되겠죠. 마이너스 기울기가 되고, 그렇다면 이 `W` 값은 사실은 최적화된 값이 여기니까 이렇게 움직여야 되겠죠. 이 값은 플러스 방향으로 움직여야 된다라는 겁니다.

그래서 우리가 식을 이야기할 때는, 이 `W` 값에다가 이 기울기를 (이 부분이 이제 우리가 수학적으로, 산술적으로 계산한 기울기죠) 즉 이 코스트 함수를 미분한 값, 즉 이 코스트 함수의 기울기가 되겠습니다. 이 기울기를 빼주면 됩니다.

$$ W := W - \alpha \frac{\partial}{\partial W} cost(W) $$

그렇죠, 그러면 여기 기울기가 플러스가 될 때는 `W`는 작아질 거고요, 이 기울기가 마이너스가 될 경우에는 (마이너스 마이너스 빼면 더해지니까) `W`가 커집니다. 그래서 이런 식으로 우리가 이제 경사하강법(Gradient Descent) 알고리즘을 적용한다는 것은, 이렇게 웨이트 값을, `W` 값을 조정한다, 이렇게 보시면 되고요.

## 경사하강법을 직접 구현하기

자, 이걸 텐서플로로 어떻게 구현할까? 이 수식을 그대로 써 주면 되겠죠. 그래서 우리 여기서 이제 알파(alpha)는 보통 우리가 학습률, 즉 러닝 레이트(learning rate)라고 하는데, 좀 작은 값 0.1 정도로 우리가 두고요.

그리고 이 부분을 우리가 잘라서 그냥 그레디언트(gradient)라는 표현을 하죠. 그레디언트라고 표현을 해서, 이 부분을 그냥 수식적으로 쭉 써 주면 되죠. `W` 곱하기 `X` 한 거에서 `Y`를 뺐죠. 그리고 그거에 곱하기 `X`가 됩니다. 그래서 이 수식을 그대로 써 주고, 이것을 평균 내 줍니다. 그래서 이런 식으로 우리가 그레디언트를 정리하고요.

```python
learning_rate = 0.1
gradient = tf.reduce_mean((W * X - Y) * X)
```

그런 다음에 이 부분, 우리가 디센트(descent)를 한다, 그래서 현재 `W` 값에다가 이 그레디언트 부분을 빼 주게 되죠. 그래서 이 `W` 마이너스, 러닝 레이트와 그레디언트 곱한 값을 빼줍니다. 그래서 새로운 `W` 값을 얻었고요.

```python
descent = W - learning_rate * gradient
```

이것을 이렇게 어사인(assign) 해야 되겠죠. 어사인 하는데, 이 텐서플로 같은 경우에는 바로 이퀄(=)로 어사인 할 수가 없고, 여기에 `assign`이라는 함수가 있습니다. 그래서 그 함수를 통해서 어사인을 할 수가 있고요, 그러면 이 값이 바뀌게 됩니다.

자, 그것을, 이 어사인하는 오퍼레이션(operation)을 우리가 `update`라는 것으로 가져왔고요. 그래서 앞으로 우리가 이 `update`를 그래프에서 바로 실행을 시키게 되면 이 동작들이 일련의 동작들이 일어나게 됩니다.

```python
update = W.assign(descent)
```

## 전체 코드 (수동 구현)

전체 코드를 한번 볼까요? 자, 우리가 이전 시간에 했던 그 리니어 리그레션(Linear Regression)과 똑같습니다. X, Y가 주어지고, `W`를 우리가 텐서플로 베리어블(Variable)로 선언하고 랜덤 값을 줬습니다. 그리고 X, Y는 우리가 실제 값을 넘겨 주려고 할 거고요, 플레이스홀더로 뒀고요. 그리고 hypothesis 그리고 코스트, 그대로 갑니다.

자, 그런 다음에 이 부분, 이전에 설명했던 부분이죠. 우리가 수동으로 이 코스트를 미니마이즈 하는 것을 만들었습니다. 러닝 레이트가 있고요, 그리고 우리가 그레디언트를 이렇게 계산했고, 그리고 이 `W` 값에다가 러닝 레이트 곱하기 그레디언트를 빼줍니다. 그래서 새로 나온 이 값을 `W`에 어사인을 해주게 되겠죠. 그래서 `W`를 업데이트 시킵니다.

```python
import tensorflow as tf

x_data = [1, 2, 3]
y_data = [1, 2, 3]

W = tf.Variable(tf.random_normal([1]), name='weight')
X = tf.placeholder(tf.float32)
Y = tf.placeholder(tf.float32)

hypothesis = X * W
cost = tf.reduce_mean(tf.square(hypothesis - Y))

# 경사하강법을 수동으로 구현
learning_rate = 0.1
gradient = tf.reduce_mean((W * X - Y) * X)
descent = W - learning_rate * gradient
update = W.assign(descent)
```

자, 이런 다음에 역시 이것이 그래프이기 때문에 세션을 열고요, 이제 베리어블을 초기화 시키고요, 그런 다음에 여기서 이 `update`라는 것을 실행을 시킵니다. 이 부분을 실행을 시킵니다. X 데이터와 Y 데이터를 던져 주면서요.

자, 그러면서 우리가 정말 이것이 잘 업데이트가 되는지, 코스트와 그리고 이 `W` 값을 봅니다. 우리가 원하는 `W` 값은 1이 되겠죠. 그래야지 `x`가 1일 때 `y`가 1이 되고 이런 식으로 되는데, 그게 잘 되는지를 한번 보는 겁니다. 그래서 이 부분은 우리가 이론 시간에 이야기했던 이 수식을 그대로 텐서플로로 그냥 적어 준 것입니다.

```python
sess = tf.Session()
sess.run(tf.global_variables_initializer())

for step in range(21):
    sess.run(update, feed_dict={X: x_data, Y: y_data})
    print(step, sess.run(cost, feed_dict={X: x_data, Y: y_data}), sess.run(W))
```

## 실행 결과 확인

자, 실행을 시켜 볼까요? 실행을 시키면 이런 형태로 쭉 나타나는데요, 첫 번째 제일 앞에 있는 게 스텝(step)이 될 거고요, 두 번째가 우리가 출력한 것이, 여기에 보시면 코스트 함수를 출력했습니다. 그래서 여기가 코스트고, 그리고 마지막에 있는 것이 `W` 값입니다.

그래서 처음에 시작할 때는 `W`가 뭐 랜덤한 값이니 1.6 이런 걸 가지겠죠. 그래서 코스트가 상당히 큰데, 이 스텝을 진행할수록 코스트가 점점 작아지고, 그리고 우리가 예상했던 대로 `W`는 1에 굉장히 가까운 수가 됩니다. 그래서 우리가 수동으로 이렇게 경사하강법(Gradient Descent) 알고리즘을 구현했는데 상당히 잘 되는 걸 보실 수가 있죠.

## 옵티마이저(Optimizer)로 간단하게 구현하기

자, 물론 우리가 텐서플로를 사용한다면, 이걸 이제 매번 우리가 이 미분 값을, 여기서 이제 이 미분 값이라는 것을 우리 함수가 좀 간단하기 때문에 쉽게 주어졌지만, 이것이 굉장히 복잡해질 수가 있죠. 이럴 때 일일이 미분할 필요가 없이, 바로 이 옵티마이저(Optimizer) 명령을 이렇게 선언하고, 그리고 이 옵티마이저를 통해서 "이 코스트를 미니마이즈 해라"고 하면, 코스트를 굳이 미분하지 않아도 자동으로 텐서플로가 이 일을 해 줍니다.

```python
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.1)
train = optimizer.minimize(cost)
```

그래서 굉장히 놀랍게도 우리가 머리 아프게 미분하지 않고도 바로 이 똑같은 일을 할 수가 있게 됩니다. 그래서 이것은 똑같은 것이고요. 자, 마찬가지입니다. 그래서 이걸 가지고 우리가 한번 실행을 시켜 보면, 이제 우리가 복잡하게 손으로 했던 것을 그냥 이 그레디언트 디센트(Gradient Descent)라는 옵티마이저를 썼고, 미니마이즈 시켰습니다.

## 다양한 초기값으로 수렴 확인

자, 그런 다음에 이제 정말 되는 걸 한번 더 보기 위해서, 텐서플로의 베리어블(Variable)에 처음에 우리가 뭐 틀린 값, 한마디로 말도 안 되는 값을 줍니다. `5.0`이라고 하면 여기 오른쪽에 `W`가 있겠죠. 처음에 우리가 여기에 `W`가 있습니다. 그래서 얘가 이렇게 정말 잘 내려가는지를 한번 실행을 시켜 보는 것이죠. 자, 코스트 등 나머지는 똑같고요, 이렇게 실행을 시킵니다.

```python
W = tf.Variable(5.0)
```

자, 한번 볼까요? 처음에 출력하는 게 이제 우리가 스텝과 `W` 값을 출력하고, 그런 다음에 한 번씩 학습을 시키게 되죠, 이 `train`이라는 것을. 자, 처음에는 우리가 `W`를 `5.0`으로 주었으니까 `5.0`이 출력되죠. 그런데 학습의 횟수를 거듭해 갈수록 빠르게 내려가서 `1.0`으로 수렴하게 됩니다. 그래서 "야, 이거 한 줄밖에 안 되는 이 그레디언트 디센트 옵티마이저가 정말 잘 동작되는구나" 하는 것을 보실 수가 있고요.

자, 다른 값으로 한번 가 볼까요? `-3` 이쪽으로 줬습니다. `W`의 처음 값이 이 값이죠. 자, 잘 될까요? 똑같은 코드인데요, 실행을 시키면 처음에는 `-3`에서 `W`가 시작하지만 여기서 이렇게 시작하겠죠. 그런데 시간이 지날수록 이 값이 계속 떨어져서 `1`까지 이렇게 `W`가 움직입니다. 자, 그래서 이 텐서플로에 있는 이 옵티마이저가 상당히 잘 동작되는구나 하는 것을 여러분들 보실 수가 있고요.

## (옵션) 그레디언트를 직접 조정하기

자, 이 부분은 이제 옵셔널(optional) 강의입니다. 여러분들이 이전까지만 이해하셔도 충분하고요. 만약에 이 텐서플로가 과연 이걸 어떻게 하는 거지 하는 걸 조금 더 이해하시고, 그걸 또 나중에 가서는 여러분들이 이 그레디언트를 조금 손대고 싶을 때가 있습니다. 내가 텐서플로가 주는 그레디언트에 뭔가 값을 좀 조정한다든지 해서, 내가 그레디언트를 임의로 조정하고 싶다, 뭐 이런 생각이 드실 수가 있는데, 그때 할 수 있습니다.

자, 마찬가지로요, 앞부분까지 다 똑같고요, 코스트까지 똑같고요, 옵티마이저가 똑같습니다. 이런 옵티마이저를 사용할 거다. 자, 그런 다음에 옵티마이저에서 바로 미니마이즈 하라고 하지 않고, 이 옵티마이저에서 `compute_gradients`로 그레디언트를 계산해 달라, 이 코스트에 맞는 걸 계산해 달라고 하면, 이 그레디언트를 계산하는 값을 돌려줍니다.

```python
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.01)

# 그레디언트를 직접 계산
gvs = optimizer.compute_gradients(cost)

# (필요하면 여기서 gvs 값을 원하는 대로 수정)

# 수정한 그레디언트를 적용
apply_gradients = optimizer.apply_gradients(gvs)
```

자, 그러면 여러분들이 이 그레디언트 계산해 준 값을 원하는 대로 좀 수정할 수가 있겠죠. 여러분들이 수정을 할 수가 있습니다. 수정을 하시고요. 그런 다음에 이 옵티마이저의 `apply_gradients`라고 해서 이 값을 어플라이(apply)를 할 수가 있습니다.

자, 지금 같은 예제에서는 그냥 `gvs`를 우리가 아무 수정 없이 바로 이렇게 어플라이를 해봤습니다. 연습용으로요. 그래서 이건 그냥 미니마이즈 한 거랑 똑같죠.

## 수동 그레디언트와 자동 그레디언트 비교

자, 그래서 우리가 비교해 보고 싶은 것이, 이 자동으로 계산된 그레디언트, 그리고 우리가 수식적으로 계산한 이것, 즉 미분의 그 수식을 그대로 놓은, 우리가 이전에 사용했던 그레디언트죠. 이게 같은 값인지를 한번 확인해 보도록 하겠습니다.

그래서 어떻게 하느냐, 이 그래프를 돌리고, 그런 다음에 이 그레디언트(우리가 수식적으로 계산한 값), 이것을 한번 출력해 보고, 그리고 `W` 값과 이 `gvs` 값을 한번 출력해 보도록 하겠습니다. 그러면서 각각 돌아갈 때마다 이 `apply_gradients` 해서, 물론 우리가 이 학습을 시켜 나가죠. 그래야 이제 이 경사도가 좀 변할, `W`가 변하고 경사도가 변하니까요.

```python
# 수식으로 직접 계산한 그레디언트
gradient = tf.reduce_mean((W * X - Y) * X) * 2

for step in range(100):
    print(step, sess.run([gradient, W, gvs], feed_dict={X: x_data, Y: y_data}))
    sess.run(apply_gradients, feed_dict={X: x_data, Y: y_data})
```

그래서 이렇게 보시면, 이게 이제 값 숫자가 좀 많아서 복잡한데요, 첫 번째 요게 스텝이 있죠. 그리고 이 두 번째가 우리가 손으로 계산한 그레디언트, 손으로 계산한 거고요. 그다음 값이 `W` 되겠죠, `W` 값 우리가 출력한 거. 그리고 이제 이게 네 번째 항목인데, 이 네 번째 항목에 이 컴퓨터가 그레디언트를 돌려 주는 것은, 이 그레디언트에 해당되는 이 웨이트 값을 이렇게 돌려 줍니다.

자, 그래서 이렇게 출력을 했는데 보시면 알겠지만, 우리가 수식적으로 계산한 값과 이 텐서플로가 자동으로 계산한 값이 같습니다. 그래서 스텝을 진행하면서 보시면 값이 거의 같습니다. 뒤에는 뭐 아마 그 소수점의 어떤 그 정밀도 때문에 조금 차이가 나겠지만 거의 같은 값이다.

그래서 우리가 이렇게 손으로 계산하는 값이 바로 이 텐서플로에서 이 옵티마이저가 만들어내는 컴퓨터 그레디언트 돌려주는 값이고, 그리고 이것을 이렇게 `apply_gradients` 할 수가 있고, 또 필요하면 여러분들 중간에 여기서 어떤 작업을 수정을 해서 할 수가 있다라는 것을, 약간 보급 과정인데 한번 이야기해 보았습니다.
