# ML lab 06-1 - TensorFlow로 Softmax Classification 구현하기

## 소프트맥스 클래시피케이션이 필요한 이유

이번 비디오에서는 소프트맥스 클래시피케이션(softmax classification)을 텐서플로(TensorFlow)를 이용해서 직접 구현해 보도록 하겠습니다. 우리가 사용하는 모든 소스코드는 여기에서 다 받으실 수가 있고요, 많은 분들이 이렇게 함께 개발해 주고 계십니다.

우리가 수업 시간에 이야기한 것처럼 이 소프트맥스(softmax) 펑션이라는 것은 여러 개의 클래스를 예측할 때 굉장히 유용한데요. 이전에 우리가 이야기했던 건 이제 바이너리 클래시피케이션(binary classification) 같은 경우에는 0이냐 1이냐 이렇게만 예측하는 것인데, 사실 실제에는 이 두 개보다 여러 개를 예측하는 경우가 많죠.

예를 들어 여러분 학점이 뭐 A, B, C, D, F 이렇게 예측하고, 그래서 여러 개의 어떤 예측할 것들이 있을 때 이 소프트맥스 펑션을 사용하면, 또 소프트맥스 클래시피케이션을 사용하면 굉장히 유용하게 됩니다.

## 소프트맥스 함수 - 스코어를 확률로

자, 이것도 마찬가지로 우리가 시작은 이런 형태에, 우리 주어진 x 값에 우리가 학습할 웨이트(weight)를 곱해서 값을 만드는 것을 시작하죠. 그런데 이렇게 나오는 값들은 그냥 이제 스코어(score)에 불과하죠. 실수에 불과하니까, 우리가 이것을 소프트맥스라고 불리는 펑션을 통과시키면 이것이 이제 확률로 나오게 됩니다.

그래서 우리가 레이블이 뭐 A, B, C 이렇게 레이블이 있다면, 이 A가 될 확률이 0.7, B가 될 확률이 0.2, C가 될 확률이 0.1 이런 식으로 우리가 확률로 표현이 가능해집니다. 이것이 이제 소프트맥스 펑션이고요.

이것의 또 하나의 특징은, 이 확률을 우리가 모든 클래스의 레이블 확률을 모두 더하게 되면 이것은 반드시 1이 될 겁니다. 자, 이런 식으로 우리가 멋지게 표현이 가능하고요.

## 텐서플로로 가설 구현하기

자, 이것을 텐서플로로 어떻게 구현할 것인가. 그것은 매우 간단합니다. 기본적으로 텐서플로로 이 수식을 그대로 써 주면 되는데요.

우리가 이런 식으로 $Y$를 표현했죠. 이것을 텐서플로로 어떻게 쓰면 될까요? 자, 이렇게 매트릭스(matrix) 곱하기, x와 W를 곱하고 그리고 바이어스(bias)를 더해 주면 되겠죠.

```python
tf.matmul(X, W) + b
```

자, 이렇게 간단하고요. 자, 그럼 이 소프트맥스 펑션, 물론 이건 이제 조금 복잡하게 생겨 있지만, 굉장히 예쁘게 생긴 펑션인데요, 이걸 직접 이렇게 구현을 하셔도 됩니다. 텐서플로를 가지고 구현해도 되지만 조금 복잡하니까, 이것을 텐서플로에서 라이브러리로 제공을 하는데, 펑션으로 제공을 하는데, `tf.nn.softmax`라고 써 주면 됩니다.

```python
hypothesis = tf.nn.softmax(tf.matmul(X, W) + b)
```

그리고 이 안쪽에 나온 값들이 스코어 값, 또는 이 값을 또 때로는 로짓(logit)이라고도 부릅니다. 로짓이라고 부르는데, 이 값을 여기다가 넣어주면 여기가 바로 소프트맥스가 나오게 됩니다. 그래서 우리가 이것이 최종 우리 가설(hypothesis)이 되겠고요, 그것이 바로 다른 말로는 어떤 레이블이 될 것인가에 대한 확률로 나오게 됩니다. 자, 이런 것이 우리가 이제 쉽게 텐서플로를 이용해서 구현 가능하고요.

## Cost 함수 - 크로스 엔트로피

자, 이 소프트맥스 펑션이 굉장히 아름다운 펑션인 것을 우리가 봤는데, 그럼 우리 여기에 걸맞은 또 로스 펑션(loss function)이 필요하죠. 그래서 로스 펑션을 우리 이렇게 정의했고, 이 로스 펑션은 텐서플로로 굉장히 쉽게 이야기할 수 있습니다.

그래서 로스 펑션을 우리가 수업 시간에 이야기한 것처럼 기본적으로 $y$ 곱하기 그리고 로그(log)의, 우리 $\hat{y}$ 즉 y 예측한 값, 또는 여기서 우리는 가설(hypothesis)이 되겠죠. 자, 이것이 우리가 바로 크로스 엔트로피(cross-entropy) 이야기를 했는데, 이것을 텐서플로로 구현하는 방법은 간단하죠. 이렇게 됩니다. 자, 그런 다음에 평균을 내는 것이죠.

자, $y$ 곱하기 그리고 로그를 사용하고 합(sum)을 합니다. 자, 그런 다음에 이 여기가 이제 여러 개의 매트릭스 값이 되겠죠. 이 곱 자체가, 그것을 합을 할 거고요. 합을 한 다음에 이제 그것을 평균을 낸다. 이 합을 하고 그다음에 이것을 평균을 내면 됩니다. 이 코스트(cost)는 그래서 합을 한 다음에 평균을 하게 되고요.

```python
cost = tf.reduce_mean(-tf.reduce_sum(Y * tf.log(hypothesis), axis=1))
```

## Cost 최소화 - Gradient Descent

자, 그런 다음에 이 코스트가 주어졌으면 우리가 이제 코스트를 미니마이즈(minimize) 해야 되겠죠. 어떻게 미니마이즈 하나요? 우리가 많이 다뤄왔던 이 경사면을 타고 내려가죠. 그래서 이 값을, 이 로스트 코스트 함수를 미분한 값에다가 앞에 러닝 레이트(learning rate)를 곱해서 그만큼 빼기를 해주면 됐죠. 기본 우리 웨이트에 이제, 이것이 기본적인 방법인데요, 계속해서 사용할 거고요.

이것을 텐서플로로 구현하는 것은 쉽게 한 줄로, 우리가 그레이디언트 디센트(Gradient Descent)라는 것을, 옵티마이저(optimizer)를 이용하고, 우리의 러닝 레이트는 이것이고, 무엇을 미니마이즈할까라는 것을 이야기해 주면 됩니다.

```python
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.1).minimize(cost)
```

자, 이렇게 이제 전체적인 것들을 구현할 수가 있고요.

## 전체 구현과 데이터 살펴보기

자, 전체 구현을 우리가 살펴보면 이렇게 한 페이지로 우리가 표시할 수가 있습니다. 그래서 설명드렸던 부분이 그대로 여기 있고요.

자, 이제 X, Y 데이터를 한번 조금 눈여겨 볼까요? x 데이터가 이제 여기는 뭐 X1, X2, X3, X4 해서 네 개의, 한 데이터가 네 개의 어떤 엘리먼트(element)가 있는 그런 형태의 데이터가 됩니다. 그래서 이런 데이터가 주어지고, 저 y 데이터를 보시면, 이전에 우리가 이제 뭐 바이너리 클래시피케이션 같은 경우에는 숫자 하나로죠, 0이나 1 뭐 이렇게 줄 수가 있지만, 여기 같은 경우에 이제 여러 개의 우리 클래스가 있기 때문에 한 자리로 표시가 불가능해집니다.

## 원핫 인코딩 (one-hot encoding)

그래서 아, 이걸 어떻게 표시할까. 우리가 뭐 여기 같은 경우에 예를 들어서 0이다, 1이다, 2다 이렇게 세 가지의 클래스가 있겠죠. 그럼 이걸 어떻게 표시할까요? 여러 가지 방법이 있는데, 그중에 우리는 원핫(one-hot)이라는 인코딩 방법을 사용합니다.

자, 원핫이라는 것은 좀 재밌는 표현인데요, 하나만 뜨겁다는 거죠. 하나만 핫하다. 그 이야기는, 자 우리가 세 개의 레이블을 가진다, 그러면 이렇게 세 자리를 만들어 줍니다. 세 자리를 만들어 주고요, 그중에 하나만 핫하게 한다. 0이면 여기만 이렇게 핫하게 한다라는 것이 되겠고요.

이런 경우 어떻게 하면 될까요? 이런 경우에는 역시 세 자리 중에 첫 번째 자리만 핫하게 해주고, 2이면 마찬가지로 세 자리 중에 두 번째 자리만 핫하게 해 준다 하는 겁니다. 그래서 핫하다는 이야기는 1이라는 이야기고요, 아닌 자리는 0이다. 그래서 이렇게 하나에만 1로 만들어 주는 것을 원핫 인코딩(one-hot encoding)이라고 하는데, 원핫 인코딩으로 표시를 했습니다.

Y 밸류를 자 볼까요? 첫 번째 이 값은 우리가 2를 나타내겠죠. 0번째, 첫 번째, 두 번째 자리이니까요. 요것도 2가 되겠고, 요것도 2가 되고, 요것은 1이 되겠죠. 자, 0번째 자리, 첫 번째 자리, 두 번째 자리 중에 첫 번째 자리가 핫하게 됐죠. 그래서 1, 요것도 1, 여기도 1, 요거는 0이겠죠. 첫 번째, 0번째 자리니까 0, 요거는 0. 자, 이렇게 레이블을 표시한 건데, 이것을 원핫이라는 방법으로 표시를 해 줍니다. 또 원핫이라는 방법으로 입력을 해 주죠.

## Placeholder의 Shape 주의점

자, 그런 다음에 우리가 X, Y에 대해서 플레이스홀더(placeholder)를 한번 생각해 볼까요? 자, 여기서 이제 셰입(shape)을 좀 주의하셔야 되는데, 이 x 데이터의 경우에는 이전과 마찬가지로 네 개의 X1, X2, X3, X4 이렇게 네 개의 피처(feature), 네 개의 엘리먼트가 주어진다, 그러면 4가 되겠고요. 앞에 None이라는 것은, 어 여러분 데이터가 몇 개나 있나요, 뭐 있는 대로 주세요, 뭐 그런 의미를 가지고 있는 것이죠.

자, y는 이제 몇 개가 될까요? 3이 됩니다. 왜냐면 이렇게 원핫으로 표시하다 보니까 이렇게 세 자리. 자, 그래서 보통 우리가 원핫으로 표시할 때는 이 y의 개수가 뭐냐면 레이블의 개수입니다. 우리가 0이냐 1이냐 2냐 세 개죠. 그렇게 하는 그 레이블의 개수, 즉 이 클래스의 개수, 이 몇 개의 클래스인가 하는 개수고요.

```python
X = tf.placeholder("float", [None, 4])
Y = tf.placeholder("float", [None, 3])
```

## Weight와 Bias의 Shape

자, 그다음에 우리가 이제 W, B 역시 정해줄 때 셰입을 한번 주의하셔야 되는데, 여기 우리가 지난번에 이야기한 것처럼, 이것은 입력되는 값 즉 x 값이 몇 개인가요? 네 개가 들어와 있었죠. 그래서 4가 되겠고, 나가는 값, 이것은 y 값인데 즉 여기서는 이 넘버 오브 클래스(number of classes)라고 할 수 있겠죠. 몇 개의 클래스가 있느냐, 3이니까 3이 되겠고요.

자, b는 y 출력값과 같죠. 그것이 세 개죠. 넘버 오브 클래스와 같습니다. 그래서 이렇게 웨이트와 바이어스를 만드실 때 이 셰입을 조금 주의하셔야 됩니다.

```python
W = tf.Variable(tf.random_normal([4, 3]), name='weight')
b = tf.Variable(tf.random_normal([3]), name='bias')
```

## 학습 시키기

자, 그렇게 되면 나머지는 이제 설명하는 것과 똑같은데요, 이 소프트맥스를 거쳐서 우리가 매트릭스 x, W 곱한 다음에 바이어스를 더하게 되고요, 그리고 코스트는 우리가 크로스 엔트로피로 이렇게 멋지게 계산했고, 자 그런 다음에 그레이디언트 디센트로 우리가 학습을 시키게 됩니다.

자, 학습하는 방법은 역시 세션(session)을 열고요, 초기화하고요, 자 그런 다음에 루프(loop)를 돌면서 옵티마이저를 실행을 시키는 거죠. 이렇게, 이때 우리가 x 데이터, y 데이터를 던져 줍니다. 자, 이렇게 학습이 일어나고, 그 중간중간 과정을 찍어 출력할 수가 있습니다.

```python
sess.run(optimizer, feed_dict={X: x_data, Y: y_data})
```

자, 이렇게 되면 우리가 소프트맥스 클래시피케이션을 학습시킬 수가 있고요.

## 예측하기 - 소프트맥스 출력

자, 그러면 우리가 이제 출력을 한번 볼까요? 어떻게 출력되는 것인가 하는 것인데, 자 조금 간단하게 한 개씩 보도록 하죠.

자, 우선 우리가 하이포시스(hypothesis)를 이렇게 주었죠. 자, 이렇게 소프트맥스를 통해서, 자 그러면 이 그림이 이제 일종의 그것을 보여 주는데요, 어떤 우리가 소프트맥스를 통해서 이 통과를 시키게 되면 이게 확률로 주어지죠. 확률로 주어졌습니다.

자, 이렇게 주어졌을 때 우리가 이것을 이용해서 그러면 새로운 값을 한번 던져 주고 "야, 너 한번 예측해 봐" 이런 거를 하고 싶은 거예요. 그래서 세션을, 우리가 학습이 끝난 다음에 이렇게 가설에 값을 던져 줍니다. 그런 다음에 예측을 하게 하죠. 자, 그러면 이 값은 이런 식으로 나올 겁니다. 소프트맥스를 통과된 확률 값으로 나오겠죠.

```python
a = sess.run(hypothesis, feed_dict={X: [[1, 11, 7, 9]]})
```

자, 그럼 여기 출력을 해 보면요, 이 값은 이런 식으로 세 개의 값이 나오죠. 자, 세 개의 값이 나왔는데, 여기는 뭐 뒤에가 조금 붙어 있어서 좀 복잡해 보이긴 하지만, 여기 보면 이게 이제 0.9 정도 되는 이야기고요, 이건 뭐 굉장히 작은 값들이라, 그래서 전체의 합은 물론 1이 될 거고요.

## argmax로 레이블 뽑기

자, 그래서 여기를 보면 이게 가장 최강자가 누군가요? 이 최강자가 (첫 번째죠). 자, 그래서 0번째 있고 첫 번째 있고 두 번째 있는데, 이 첫 번째 있는 게 최강자다 하는 것을 정해야 되는데, 이 값에서 이 첫 번째 있는 것을, 이제 뭐 0, 1 이렇게 나오면 우리의 예측하는 것은 이게 아니라 얘를 선택하는 거였죠.

자, 이렇게 해야 되는데 이것을 자동으로 해 주는 것이 또 텐서플로에 있는 아그맥스(argmax)라는 것입니다. 그래서 아그맥스에 이런 형태를 주면, 몇 번째 있는 아규먼트(argument)가, 0번째, 첫 번째, 두 번째 있는 것이 가장 높은가요를 물어보는 겁니다. 그래서 이것이 바로 1을 돌려주게 될 겁니다.

자, 이 축(axis)이라는 것은 어떤 축을 기준으로 할 것인가. 제일 안쪽에 있는 거 첫 번째 축이죠, 0, 첫 번째 엑시스(axis). 그래서 여기 있는 것 중에 최강자를 골라주세요 하면 1을 돌려주게 됩니다.

```python
print(sess.run(tf.arg_max(a, 1)))
```

자, 이렇게 우리가 예측을 해 볼 수 있고, 돌아온 소프트맥스 값을 우리가 원하는 레이블로 바꿀 때는 여기서 아그맥스를 사용한다는 것을 이야기 드렸습니다.

## 여러 개를 한 번에 예측하기

자, 물론 우리가 이제 질문을 할 때 한 개씩 질문하면 조금 심심하니까 여러 개를 동시에 질문할 수도 있죠. 자, 이렇게 여러 개를 x로, "이런 경우에는 어떻니, 이런 경우에는 어떻니, 이런 경우에 어떻니"라고 질문을 합니다. 자, 그러면 우리 친절한 텐서플로가 답을 다 알려 주겠죠.

```python
all = sess.run(hypothesis, feed_dict={X: [[1, 11, 7, 9],
                                          [1, 3, 4, 3],
                                          [1, 1, 0, 1]]})
print(all, sess.run(tf.arg_max(all, 1)))
```

그래서 `all` 하면 이런 형태로, 우리 확률만 보면 이런 형태로 확률로 돌려줍니다. 그래서 이게 첫 번째 질문에 대한 답이고, 두 번째에 대한 질문의 답, 그리고 이 마지막이 세 번째에 대한 질문의 답이 될 겁니다.

자, 그럼 뭐 숫자가 좀 복잡해지는데요, 한번 보면 이게 이제 0.9 정도 되는 것이고 대부분 얘가 이제 성능이네요, 여기서는. 그리고 여기를 한번 볼까요? 보면 여기도 얘가 이제 0.9 정도 되는 거네요. 그래서 여기가 승리하고, 여기는 누군가요? 여기도 얘가 승리하네요, 0.9. 자, 나머지 값은 좀 작죠.

자, 이렇게 되었습니다. 그런 다음에 우리가 알고 싶은 것은 이제 레이블을 구체적으로 말해 보라, 0이냐 1이냐 2냐 할 때는 바로 아그맥스를 쓴다고 했죠. 그런 경우에 우리가 첫 번째 거기에 해당되는 최강자를 찾아주세요 하면, 첫 번째 여기는 1이 되겠고요, 여기는 0, 여기는 2 해서, 우리 답이 1, 0, 2 이렇게 돌아옵니다.

자, 이렇게 우리가 이제 소프트맥스로 한번 구현도 하고, 또 아그맥스를 사용해서 그 나온 값을 실제 레이블로 한번 바꿔 보았습니다.
