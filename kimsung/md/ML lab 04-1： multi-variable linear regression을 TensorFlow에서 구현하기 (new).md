# ML lab 04-1 - multi-variable linear regression을 TensorFlow에서 구현하기

이번 비디오에서는 멀티 베리어블(multi-variable) 리니어 리그레션(linear regression)을 텐서플로(TensorFlow)를 이용해서 직접 구현해 보도록 하겠습니다.

지난번 시간까지 우리가 했던 실습은 그냥 x 하나 있고 y 하나 있는, 사실상은 굉장히 토이 프로젝트(toy project), 장난감 프로젝트라고 할 수 있는데요. 이제 멀티 베리어블을 사용할 수가 있으면 실제로 우리가 가지고 있는 데이터에 적용해 볼 수가 있게 됩니다.

이 비디오에 사용한 모든 소스 코드는 마찬가지로 깃허브(GitHub)에서 다 다운받으실 수가 있고요, 많은 분들과 함께 개발하고 있습니다.

## 예제 문제: 세 개의 입력으로 기말고사 점수 예측하기

자, 우선 우리가 이제 이론 시간에 이야기한 것처럼, 여기 같은 경우는 x가 세 개, x 데이터가 세 개입니다. `x1`, `x2`, `x3` 이렇게 되고요. 이제 이걸 가지고 y를 예측하는데, 여기는 이제 중간고사 점수가 있죠. 중간고사 점수 첫 번째, 두 번째, 세 번째면, 마지막 기말고사가 얼마일까 예측하는 그런 모델을 만들 텐데요.

이렇게 보시는 것처럼 이제 x가 세 개입니다. 사실상은 이 x가 더 많을수록 우리가 더 잘 예측할 수 있겠죠. 예를 들면 이전에 출석을 몇 번 했는지, 또는 다른 중간에 퀴즈 같은 게 있으면 퀴즈 점수는 어떤지, 또는 다른 이전의 과목들 점수는 어떤지 해서, 이것들이 이제 굉장히 더 많이 늘어날 거고요. 이것이 이제 늘어날수록 이거를 예측하는 힘은 더 세질 겁니다. 그래서 프리딕션 파워(prediction power)가 세진다, 그런 말을 가끔 하는데요.

자, 우리가 이런 경우에, 이 예제에서는 세 개만 있다고 가정을 합시다.

## 가설(Hypothesis) 세우기

자, 그럼 어떻게 하면 좋을까요? 우리 하이포시스(hypothesis)는 자연스럽게, 기존에는 `x*w` 하나밖에 없었지만 이거 세 개로, 이제 우리가 학습해야 될 웨이트(weight)도 세 개다, 이렇게 생각하시면 되겠죠. 그래서 이렇게 세 개를 쭉 늘어서 주시면 됩니다.

$$ H(x_1, x_2, x_3) = x_1 w_1 + x_2 w_2 + x_3 w_3 $$

여기서 이제 바이어스(bias)는 우리가 좀 간략하게 하기 위해서 잠깐 생략을 했고요.

## 텐서플로 코드로 작성하기 (변수 나열 방식)

자, 이것을 텐서플로 코드로 한번 써 볼까요? 자 이렇게 됩니다.

우선 여기 데이터를 좀 정의해 보죠. y 데이터를 우선 정의해 볼까요? 이렇게 똑같이 152, 185 이런 순으로 값을 썼고요. 자 이것 이제 x, `x1` 데이터, `x2`, `x3` 데이터를 이렇게 쭉 나열을 해 줍니다. 이런 식으로 썼습니다.

```python
x1_data = [...]
x2_data = [...]
x3_data = [...]
y_data  = [152., 185., ...]
```

자 그런 다음에 우리가 이제 이걸 또 뭐 피드 딕트(feed_dict) 같은 걸로 던져 주기 위해서 이렇게 플레이스홀더(placeholder)를 만들고요.

```python
x1 = tf.placeholder(tf.float32)
x2 = tf.placeholder(tf.float32)
x3 = tf.placeholder(tf.float32)
Y  = tf.placeholder(tf.float32)
```

자 그런 다음에 이제 이 웨이트(weight)들은 뭐 각각 하나의 값이죠. 그래서 마찬가지로 이전에 하나의 값으로 이렇게 줍니다. 자 여기 바이어스(bias)도 여기 뭐 마찬가지로 주고요.

```python
w1 = tf.Variable(tf.random_normal([1]))
w2 = tf.Variable(tf.random_normal([1]))
w3 = tf.Variable(tf.random_normal([1]))
b  = tf.Variable(tf.random_normal([1]))
```

그런 다음에 우리 하이포시스(hypothesis)는 이렇게 쭉 차례로 이 식 그대로 곱하기를 해서 더하죠. 자 다 한 다음에는 여기는 이제 우리가 바이어스를 추가했습니다. 그래서 이렇게 하이포시스를 하고요.

```python
hypothesis = x1 * w1 + x2 * w2 + x3 * w3 + b
```

## 비용(Cost)과 옵티마이저

자 이렇게 하이포시스가 있으면 마찬가지로 우리 코스트(cost)는 이런 식으로, 하이포시스와 우리 y 차의 제곱을 평균한 거, 이전과 마찬가지고요.

```python
cost = tf.reduce_mean(tf.square(hypothesis - Y))
```

우리 옵티마이저(optimizer)를 사용할 거고, 가끔 이제 뭐 숫자가 0.01, 0.001 이렇게 주기가 복잡해지니 1e-5(1 곱하기 2의 -5승 같은 식)로 이렇게 줄 수도 있습니다. 그래서 이렇게 러닝 레이트(learning rate)를 낮게 주었고요.

```python
optimizer = tf.train.GradientDescentOptimizer(learning_rate=1e-5)
train = optimizer.minimize(cost)
```

## 학습시키기

자 그런 다음에 똑같은 방법으로 이렇게 세션(session)을 만들고, 베리어블 이니셜라이즈(variables initializer)를 하고, 자 학습을 시킵니다.

```python
sess = tf.Session()
sess.run(tf.global_variables_initializer())
```

자 이 학습을 시킬 때 자세히 보시면, 이 `train`을 결국은 돌리게 될 거고요. 그때 같이 우리가 관심 있게 코스트(cost)와 하이포시스 밸류(hypothesis value)를 한번 같이 기록을 했다가 한번 출력을 해 보는 그런 모델이죠.

자 이때 이 피드 딕트(feed_dict)로 우리가 이 값들, `x1`, `x2`, `x3`를 던져 주게 됩니다.

```python
for step in range(2001):
    cost_val, hy_val, _ = sess.run(
        [cost, hypothesis, train],
        feed_dict={x1: x1_data, x2: x2_data, x3: x3_data, Y: y_data})
```

자 이전과 똑같고요, 이 부분만, 데이터 부분 그리고 이 x 부분, 웨이트 부분만 좀 확장되었다, 이제 말이 늘어났으니 이렇게 보시면 됩니다.

## 실행 결과

자 그러면 실제를 한번 보면요, 이렇게 우리가 쭉 이제 뭐 예측하는 것을 이렇게 쭉 출력하면서 보면, 프리딕션(prediction)과 코스트를 출력해 봤는데, 처음에는 코스트가 상당히 높죠. 그러다가 이제 많이 실행을 시킬수록 이 코스트가 낮아집니다.

낮아지면서 이 값이 우리가, 이 값을 한번 볼까요? 마지막에 이 값이 우리가 최종 y 값은 이건데요, 우리가 목표로 하는 y 값이 이거인데, 이 값과 똑같지 않지만 유사하게 수렴한다고 볼 수가 있겠죠. 이 그림이 좀 복잡해졌네요. 자 152가 148, 또 185가 186, 이렇게 우리가 거의 수렴하게 간다라고 볼 수가 있습니다.

## 변수 나열 방식의 한계

자 이제 그림이 굉장히 복잡해졌는데요. 이 이유 중에 하나가, 이 보시면 이 뭐 굉장히 복잡하죠. 그래서 뭔가 좀 아름답지가 않습니다. 그리고 이제 이 x 데이터가 여기 세 개밖에 없지만, 이게 뭐 100개가 된다면 이 코드는 아 정말 마치 스파게티 코드(spaghetti code) 같이 복잡해질 거죠. 그래서 이런 방법은 이제는 사용하지 않는다라고 보시면 좋고요.

## 행렬(matrix)로 대체하기

자 그것을 대체할 것이 바로 우리가 좋아하는 행렬(matrix)이 되겠는데요. 이 행렬은 이 x 개수가 얼마든지 상관없이 이렇게 쭉 옆으로 행렬로 나열해 주고, 거기에 개수에 맞게 웨이트만 써 주면, 이것을 그냥 곱하고 하면 한 번에 다 곱해서 값을 내게 됩니다.

그래서 우리가 행렬을 이렇게 `XW` 행렬곱으로 표시를 했고요.

$$ H(X) = XW $$

## 행렬 방식 구현: 데이터 정의

자 이것을 한번 구현해 볼까요? 자 이제는 이것이 이제 행렬이기 때문에, 그냥 이 값을 이렇게 각각 하나가 `x1`, `x2`, `x3` 값을 이렇게 넣어 주고요. 그래서 요것이 하나의, 어떻게 보면 엘리먼트(element) 하나의 인스턴스(instance), 하나의 x 데이터가 되겠고요. 여기에 해당되는 y 데이터가 y 값이 되겠습니다. 이렇게 맵핑(mapping)이 되고요.

두 번째 x 데이터가 세 개 값이 되겠죠. `x1`, `x2`, `x3` 이렇게 되고 여기에 맵핑하는 값이 이렇게 되고, 마지막에 마찬가지로 이 세 개, 그리고 매핑하는 값이 이렇게. 그래서 만약에 x가 뭐 많아졌다 그러면 여기에 항들은 조금 더 추가가 되면 되는 것이죠. 전체는 같은 구조입니다.

```python
x_data = [[73., 80., 75.],
          [93., 88., 93.],
          ...]
y_data = [[152.],
          [185.],
          ...]
```

## 행렬 방식 구현: 플레이스홀더와 shape

자 그런 다음에 이제 플레이스홀더(placeholder)를 우리가 만들어야 되겠죠. X, Y 만들 때 조금 주의해 보셔야 될 게 이제 쉐입(shape)입니다.

자 여기는 우리가 `x1`, `x2`, `x3`, 이게 세 개가 들어갔죠. 그러다 보니까 이 뒤에 있는 값은 세 개가 됩니다. 3이라고 우리가 잡아 줄 거고요. 그리고 앞에 있는 이 부분은 이 데이터가 몇 개냐 하는 것입니다. 지금은 뭐 다섯 개로 정해져 있지만 우리가 필요에 따라 더 추가할 수가 있겠죠, x 데이터를. 그래서 이거는 대략적으로 x 데이터는 n개를 가질 것이다.

그래서 이런 경우에는 이 텐서플로에서는 `None`이라고 표시를 해 줍니다. 그 말은 네가 원하는 만큼 줄 수가 있다라는 이야기고요. 그 대신에 각 엘리먼트가 가지는 값은 세 개다, 그것은 정해져 있습니다. 이렇게 되고요.

```python
X = tf.placeholder(tf.float32, shape=[None, 3])
Y = tf.placeholder(tf.float32, shape=[None, 1])
```

마찬가지로 예측 y, 이거 보시면 뭐 값이 하나로 예측되죠. 우리가 예측하는 것이 이제 마지막 그 최종 파이널(final) 그 하나의 점수, 그러니까 이게 하나가 되고요. 역시 마찬가지로 이것은 n개를 우리가 예측하겠다. 행렬로 보시면 이게 이렇게 쭉 늘어나는 거고요, n개로. 자 이렇게 늘어나게 되고 그럼 이 결과도 이렇게 n개로 늘어나게 됩니다. n은 그대로 가겠죠. 자 그렇게 됩니다.

## 행렬 방식 구현: 가설, 비용, 학습

그런 다음에 우리가 하이포시스(hypothesis)를, 이제 행렬곱을 그대로 가고요.

```python
W = tf.Variable(tf.random_normal([3, 1]))
b = tf.Variable(tf.random_normal([1]))

hypothesis = tf.matmul(X, W) + b
```

자 그런 다음에 우리 이전에 봤던 것처럼 역시 코스트(cost)가 똑같고요. 이제 옵티마이저(optimizer)도 똑같습니다. 그리고 학습하는 모든 방법은 똑같고요.

```python
cost = tf.reduce_mean(tf.square(hypothesis - Y))
optimizer = tf.train.GradientDescentOptimizer(learning_rate=1e-5)
train = optimizer.minimize(cost)
```

자 이 부분을 한번 보면, 이전에 굉장히 복잡하게 나열되어 있던 것에서 굉장히 단순하게 간단해졌습니다. 이 데이터만 뭐 많으니까 복잡하게 볼 수도 있겠지만 나머지 부분은 굉장히 간단해졌습니다. 그래서 이것이 바로 이 행렬(matrix)의 장점이고요. 그 결과는 여러분들 보시면 알겠지만 이전과 똑같이 나옵니다.

## 정리

자 그래서 우리가 이렇게 행렬을 사용하게 되면, 이 베리어블(variable) 개수가 몇 개 있든지 상관없이 우리가 원하는 대로, 원하는 개수만큼, 그리고 그것을 굉장히 쉽게 이렇게 바로 표현해서 우리가 구현할 수 있다는 것을 말씀드렸습니다.

자 다음 시간에는, 우리가 이게 굉장히 이제 많은 데이터를 다루게 될 텐데, 이걸 일일이 이 파일에다가, 소스 코드에다가 적어 주기가 힘드니까, 이 데이터를 파일에다가 적어 주고 그것을 로딩(loading)하는 방법에 대해서 이야기해 보도록 하겠습니다.
