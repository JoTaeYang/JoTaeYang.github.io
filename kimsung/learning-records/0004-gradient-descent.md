## 다룬 것 (lec 03 기반)
- cost가 W에 대해 **밥그릇(U자)** 모양임을 (1,1)(2,2)(3,3) 데이터로 보임: cost(0)≈4.67, cost(1)=0, cost(2)≈4.67.
- 경사하강법 = 안대 쓰고 한라산 내려오기. 아무 W에서 시작 → 기울기 재고 → 내리막 한 발짝 → 반복 → 바닥.
- **두 손잡이**: ① 기울기(gradient)=방향, ② 학습률(α)=보폭. α 너무 크면 튕김/너무 작으면 느림.
- 업데이트식 딱 한 줄만: `새 W ← 지금 W − α×(기울기)`. "−"=하강, 미분=기울기 재는 자(컴퓨터가 대신).
- 바닥 근처에서 발짝이 저절로 작아짐(기울기가 평평→발짝 작아짐) = 그림의 촘촘한 공으로 시각화.
- 볼록 함수(convex): 웅덩이 하나 → 시작점 무관 항상 최저점 보장. 실무 주의점("cost가 밥그릇인가 확인")으로 착지.

## 교수법 결정
- 수학 공포 대응 원칙 유지: 미분을 **손으로 안 시킴**. "기울기를 재는 자"로만 직관화, 실제 미분 유도식(lec의
  ∂/∂W, 2/2m 소거)은 **의도적으로 뺌**. 업데이트식도 원본의 Σ·x^(i) 형태 대신 `W ← W − α×(기울기)`로 탈수식화.
- b를 뺀 단순화(H=Wx)를 그대로 채택 — 1차원 밥그릇이라 그림 한 장으로 설명 가능.
- L3(코드)과 명시적 콜백: `learning_rate=0.01`, `GradientDescentOptimizer().minimize(cost)`가 오늘 이 한 줄이라고
  연결. 개념↔코드 인터리빙 리듬(노트 참조) 계속 지킴.
- `.eq` 스타일을 assets/style.css로 **승격**(레슨 2에 이어 2번째 사용 = 재사용 신호). 레슨 2 인라인은 그대로 둠(무해).
- 밥그릇+굴러내리는 공 SVG는 레슨 전용 소품(재사용성 낮아 assets 미승격). 발짝이 바닥서 촘촘해지는 것까지 그림에 반영.
- 퀴즈 3개: 정답 인덱스 0/2/1로 분산, 보기 단어 수 최대한 균등(방향/보폭/위치 등).
