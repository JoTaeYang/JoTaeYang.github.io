# 0006 — 레슨 6: 여러 입력 (multi-variable, lec 04)

_기록일: 2026-07-22_

## 계기
Phase 1 완주 후 "다음 레슨 진행". Phase 2 진입점 = lec 04(multi-variable). 개념(lec) 레슨.

## 다룬 것 (lec 04 기반)
- 동기: 입력 하나(공부시간)는 장난감. 현실은 피처 여럿 → 퀴즈1·2·중간 → 기말 예측(실제 데이터 표).
- 핵심 안심 메시지: 선형회귀 3부분 중 **cost·경사하강법은 그대로, 가설만 넓힘.** `H=w₁x₁+w₂x₂+w₃x₃+b`.
- 피처 100개면 식이 길어짐 → **행렬로 접기.** 행렬 = "숫자 표", 쓸 건 행렬 곱(가로·세로 짝지어 곱해 더하기) 하나뿐.
- 행렬 곱으로 긴 가설이 그대로 복원됨을 시각화 → `H(X)=XW`. XW 순서는 구현 관례(값은 Wx와 동일, 대문자=행렬).
- 진짜 이점: 인스턴스 여러 개(5명)를 X로 쌓으면 W 그대로, 한 번 곱으로 전부 예측. 만 명이어도 코드 불변.
- shape 규칙: `[5,3]×[3,1]=[5,1]`, 안쪽 맞고 바깥이 결과. **W=[피처수, 출력수]**, 인스턴스는 None(TF)/-1(넘파이).
- 출력 여러 개 미리보기: `[n,3]×[3,2]=[n,2]` → 신경망 예고.

## 교수법 결정
- 수학 공포 관리: **행렬을 "표 곱하기"로 탈신비화.** 손으로 행렬 곱 안 시킴 — 한 줄짜리 시각화(.mat) 하나로 직관만.
  shape는 색으로 안쪽(맞춤)/바깥(결과) 구분하는 .shape 소품으로 규칙만 보여줌.
- 실무 훅 강조(미션=커리어): "코드에서 shape=[None,3]의 None이 이거다", "W 크기 거꾸로 구하기"를 key 콜아웃으로.
  → L7(lab 04-1) 코드에서 바로 회수될 포석.
- lec 원본의 데이터 값(73/80/75→152 등)·shape 예시 그대로 사용, 없는 내용 추가 없음.
- 소품(.mat/.shape/.feat)은 이 레슨 전용(재사용성 낮아 assets 미승격). `.eq`는 기존 공용 재사용.

## 용어집 갱신
- 새 그룹 `#multi`: 다변수 선형회귀 / 행렬·행렬곱(XW) / 인스턴스 / 셰이프([행,열], None·-1). 레슨에서 이 anchor로 링크.
