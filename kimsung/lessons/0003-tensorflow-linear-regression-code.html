0. 먼저: Python·TensorFlow 30초 오리엔테이션

Python은 프로그래밍 언어, TensorFlow는 구글이 만든 머신러닝 도구상자(라이브러리)입니다. 코드에서 tf.로 시작하면 "TensorFlow 도구를 꺼내 쓴다"는 뜻이에요. 이 코드를 읽는 데 필요한 Python은 이게 전부입니다:

a = 3a라는 이름에 3을 담는다 (대입).
[1, 2, 3]값 세 개가 든 목록(리스트).
square(x)이름(...) — 도구를 호출하고 괄호 안 값을 넘긴다.
for step in range(2001):0부터 2000까지 2001번 반복한다.
if step % 20 == 0:step이 20의 배수일 때만 (즉 가끔) 실행.
print(...)괄호 안 값을 화면에 출력.

이제 코드를 다섯 조각으로 끊어 읽습니다. 각 조각은 레슨 2에서 이미 배운 개념이에요.

조각 1 — 데이터
x_train = [1, 2, 3]
y_train = [1, 2, 3]

레슨 2의 그 데이터 그대로예요: (1,1) (2,2) (3,3). x는 입력(피처), y는 정답.

조각 2 — 찾아야 할 값 W와 b
W = tf.Variable(tf.random_normal([1]), name='weight')
b = tf.Variable(tf.random_normal([1]), name='bias')

W(기울기)와 b(y절편)를 Variable로 선언합니다. Variable은 "텐서플로가 학습하면서 스스로 바꾸는 값"이라는 뜻이에요. 처음엔 정답을 모르니 random_normal로 아무 랜덤 값이나 넣고 시작합니다. 학습이 이걸 바른 값으로 밀어줄 거예요.

조각 3 — 가설 H(x) = Wx + b
hypothesis = x_train * W + b
이건 레슨 2의 바로 그것 x * W + b = H(x) = Wx + b. 글자 그대로 옮긴 거예요. 이 선이 내놓는 예측.
조각 4 — 비용 cost
cost = tf.reduce_mean(tf.square(hypothesis - y_train))

레슨 2의 cost 공식을 안쪽부터 읽어보세요:

  • hypothesis - y_train → 오차 (예측 − 실제)
  • tf.square(...) → 오차를 제곱 (부호 없애고 큰 오차에 벌점)
  • tf.reduce_mean(...) → 전부 평균 → 숫자 하나 = cost
조각 5 — cost 최소화
optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.01)
train = optimizer.minimize(cost)

레슨 2 마지막의 "cost를 가장 작게 만드는 W, b를 찾아라"가 이 두 줄입니다. 방법은 경사하강법(Gradient Descent) — 다음 레슨에서 "안대 쓰고 언덕 내려가기"로 풀어드릴게요. 지금은 "cost를 줄여주는 마법 버튼" 정도로 봐도 충분합니다.1 learning_rate(보폭)만 눈에 담아두세요.

실행 — 세션 열고 2001번 반복 학습
sess = tf.Session()
sess.run(tf.global_variables_initializer())   # Variable 초기화(필수)
for step in range(2001):
    sess.run(train)                            # ← 이 줄이 돌 때마다 W,b가 한 걸음 개선
    if step % 20 == 0:
        print(step, sess.run(cost), sess.run(W), sess.run(b))

TensorFlow는 코드를 짜자마자 계산하지 않고, 먼저 그래프(설계도)를 만든 뒤 sess.run(...)으로 "실행" 버튼을 눌러야 돌아갑니다. train을 2001번 실행하면 그때마다 W와 b가 조금씩 좋아져요.1

예상 출력 — cost는 0으로, W는 1로, b는 0으로

실행하면 이런 흐름이 찍힙니다. (랜덤 시작값이라 숫자는 매번 조금씩 달라도, 수렴하는 방향은 항상 같아요.)

step    cost       W       b 0    2.09    [1.645]  [ 0.057] 20    0.36    [1.300]  [-0.050] 100    0.051   [1.130]  [-0.060] 500    0.0015  [1.030]  [-0.030] 2000    0.00001 [1.004]  [-0.009]

읽는 법: 맨 왼쪽은 학습 횟수. 처음엔 랜덤이라 cost가 크고 W·b가 엉뚱하죠(2.09). 그런데 반복할수록 cost가 0으로 쪼그라들고, W는 1에, b는 0에 착 달라붙습니다.1 우리가 눈으로 "정답은 W=1, b=0"이라 했던 그 선을, 기계가 스스로 찾아낸 거예요. 이게 "학습됐다"의 실체입니다.

보너스 — 학습된 모델로 예측하기

데이터를 조금 바꿔 (1,2.1)(2,3.1)(3,4.1)(4,5.1)(5,6.1)로 학습시키면, 정답 선은 W=1, b=1.1. 학습 뒤 새 x를 넣어 예측하면:

print(sess.run(hypothesis, feed_dict={X: [5]}))
print(sess.run(hypothesis, feed_dict={X: [2.5]}))
print(sess.run(hypothesis, feed_dict={X: [1.5, 3.5]}))
[6.1006]    # 5 × 1 + 1.1 ≈ 6.1 [3.5998]    # 2.5 × 1 + 1.1 ≈ 3.6 [2.5997 4.5999] # 두 개 동시에

학습 데이터에 없던 x=5, 2.5도 그럴듯하게 맞힙니다 — 이게 회귀 모델을 "쓴다"는 것. 자세한 골격은 📄 TF 선형회귀 치트시트에 정리해뒀어요(다음 랩에서 계속 씁니다).
