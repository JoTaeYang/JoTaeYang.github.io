레슨 6 · Phase 2 — 여러 입력

입력이 여러 개일 때:
multi-variable

지금까진 입력이 딱 하나였죠(공부 시간 → 점수). 현실은 그렇지 않아요. 오늘은 입력을 여러 개로 늘리고, 그걸 행렬 하나로 깔끔하게 접는 법을 봅니다. 행렬, 겁내지 마세요 — 표 곱하기일 뿐이에요.

이 레슨을 마치면 할 수 있는 것 "입력이 3개면 가설은 w₁x₁+w₂x₂+w₃x₃+b", 그리고 "그걸 짧게 쓰면 H(X) = XW" — 이 두 문장을 퀴즈 점수로 파이널을 예측하는 실제 상황에 대고 설명할 수 있습니다.

1. 현실엔 입력이 하나가 아니다

레슨 2~5에서 우리는 입력이 하나인 세상에 있었어요 — "공부 시간 x → 점수 y". 그런데 실무 데이터는 거의 항상 입력이 여럿입니다.1

강의의 실제 예시: 퀴즈1·퀴즈2·중간고사 점수로 기말(final) 점수를 예측하기. 입력이 세 개죠.

x₁ (퀴즈1)x₂ (퀴즈2)x₃ (중간)y (기말)
738075152
938893185
899190180

기말을 안 보고도 세 점수만으로 기말을 맞힐 수 있다면 좋겠죠? 이렇게 예측에 쓰는 입력 하나하나를 피처(feature)라고 불러요. 여기선 피처가 3개입니다.

2. 좋은 소식: 바뀌는 건 가설 하나뿐

레슨 2~5에서 선형 회귀는 세 부분이었죠 — ① 가설 H, ② cost, ③ 경사하강법. 입력이 여러 개가 돼도 ② cost와 ③ 경사하강법은 그대로예요. 딱 ① 가설만 넓히면 됩니다.1

입력이 하나일 땐 H(x) = wx + b였죠. 입력이 셋이면? 피처마다 weight를 하나씩 붙여서 옆으로 늘려요:

H(x₁,x₂,x₃) = w₁x₁ + w₂x₂ + w₃x₃ + b

각 w는 "그 피처가 결과에 얼마나 중요한가"예요. 학습이 이 w₁,w₂,w₃(과 b)를 데이터에서 찾아냅니다. 피처가 100개면? 똑같이 w₁x₁ + … + w₁₀₀x₁₀₀ + b로 쭉 늘리면 돼요.

3. 문제: 100개를 이렇게 쓰면 손이 아프다

그런데 피처가 100개, 200개가 되면 이 식이 끔찍하게 길어집니다. 매번 항을 100개씩 쓸 순 없죠. 그래서 이걸 한 덩어리로 접는 도구가 행렬(matrix)이에요.1

행렬은 그냥 "숫자 표" 행렬이라고 하면 복잡해 보이지만, 숫자를 네모나게 늘어놓은 표일 뿐이에요. 우리가 쓸 건 딱 하나 — 행렬 곱셈: "가로 줄과 세로 줄을 짝지어 곱해서 더하기".

4. 행렬 곱으로 가설을 한 줄에

피처 x₁,x₂,x₃를 가로로, weight w₁,w₂,w₃를 세로로 세워 곱하면:1

x₁  x₂  x₃ × w₁
w₂
w₃ = x₁w₁ + x₂w₂ + x₃w₃

보세요 — 짝지어 곱해 더했더니(x₁·w₁ + x₂·w₂ + x₃·w₃), 2번의 그 긴 가설이 그대로 다시 나왔어요. 이걸 대문자로 X(피처들), W(weight들)라 쓰면, 가설 전체가 딱 이 한 줄이 됩니다:

H(X) = XW
왜 wx가 아니라 XW 순서예요? 이론·강의에선 Wx라 쓰지만, 실제 코드에선 X를 앞에 둔 XW로 씁니다. 행렬은 순서로 크기를 맞춰야 곱셈이 되거든요. 곱셈 값은 Wx와 똑같아요 — 순서만 실무 관례입니다. (대문자 X·W는 "이건 행렬이다"라는 표시.)1

5. 진짜 마법: 데이터가 만 명이어도 한 번에

행렬의 놀라운 점이 여기 있어요. 학생이 1명이 아니라 5명이면(=인스턴스 5개), 피처 표 X를 아래로 쌓기만 하면 됩니다. 그런데 W는 그대로예요. 그리고 곱하면 5명의 예측이 한 번에 쏟아집니다.1

[5, 3] × [3, 1] = [5, 1]

읽는 법 — [행, 열]은 표의 크기(shape)예요:

  • X = [5, 3]: 학생 5명(인스턴스) × 피처 3개.
  • W = [3, 1]: 피처 3개를 받아 결과 1개(기말 점수).
  • 가운데 3·3이 맞아야 곱셈이 되고, 바깥의 5·1이 결과 크기 → [5, 1](5명 각자의 예측).

학생이 만 명이어도 만 번 계산할 필요 없이, X를 [10000, 3]으로 쌓아 한 번 곱하면 끝. 이게 실무에서 행렬을 쓰는 진짜 이유예요 — 데이터가 많아도 코드는 안 바뀝니다.

실무에서 바로 만날 장면 — W 크기 거꾸로 구하기 X(피처 수)와 Y(출력 수)는 문제가 정해줘요. 우리가 정할 건 W의 크기인데, 규칙은 간단합니다: W = [피처 수, 출력 수]. 인스턴스 개수는 신경 안 써요(가변이니까). 코드에서 shape=[None, 3]의 None이 바로 "인스턴스는 몇 개든"이라는 뜻입니다(넘파이는 -1).1

6. (미리 보기) 출력이 여러 개여도 똑같다

출력을 1개가 아니라 2개로 내고 싶으면? W의 뒷자리만 바꾸면 돼요. 이건 나중에 신경망(neural network)에서 다시 만납니다.1

[n, 3] × [3, 2] = [n, 2]

피처 3개 → 출력 2개면 W = [3, 2]. 규칙은 5번과 완전히 같아요. 그래서 행렬 하나로 입력이 여럿이든, 데이터가 많든, 출력이 여럿이든 전부 처리됩니다.
