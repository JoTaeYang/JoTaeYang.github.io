레슨 2 · Phase 1 — 선형 회귀

선을 긋는다는 것:
가설과 cost

"머신이 학습한다"는 말이 실제로 무슨 일인지, 오늘 한 장면으로 봅니다. 수식이 세 개 나오지만, 전부 말로 먼저 풀어드릴게요. 겁먹지 마세요.

이 레슨을 마치면 할 수 있는 것 누가 "머신러닝 모델이 학습한다는 게 뭐예요?"라고 물으면 이렇게 답할 수 있습니다 — "데이터에 가장 잘 맞는 선을 찾는 거예요. 틀린 정도(cost)를 가장 작게 만드는 선이요."

1. 선형 회귀 = "데이터에 가장 잘 맞는 직선 찾기"

레슨 1에서 "답이 넓은 숫자면 회귀"라고 했죠. 그중 가장 기본이자 실무에서 제일 많이 쓰이는 게 선형 회귀(Linear Regression)입니다.1

세상 많은 일이 "직선"으로 대충 설명됩니다 — 공부를 많이 할수록 점수가 오르고, 집이 클수록 값이 비싸고, 광고비를 더 쓸수록 매출이 오르고. 이렇게 입력이 커지면 출력도 일정하게 커지는 관계를 직선 하나로 그려보자는 게 선형 회귀입니다.1

아주 간단한 데이터를 봅시다. 공부 시간 x와 점수 y가 이렇게 있어요: (1, 1), (2, 2), (3, 3). 점 세 개를 찍으면 딱 일직선으로 늘어섭니다.

2. 가설(Hypothesis): 선을 수식 하나로 적기

"이 데이터엔 어떤 직선이 맞을 거야"라고 미리 세우는 짐작을 가설(Hypothesis)이라고 부릅니다. 직선 하나는 딱 두 개의 숫자로 정해져요:1

H(x) = W·x + b
  • W (기울기, weight) — 선이 얼마나 가파른가. x가 1 늘 때 y가 얼마나 오르는지.
  • b (y절편, bias) — 선이 세로로 어디쯤 걸려 있는가. x가 0일 때의 y값.

H(x)는 "이 선이 내놓는 예측값"이라는 뜻이에요(H는 Hypothesis의 H). W와 b를 바꾸면 선이 이리저리 기울고 오르내립니다. 예를 들어 H(x) = 1·x + 0 은 위 그림의 딱 맞는 선이고, H(x) = 0.5·x + 2 는 완만하게 뜬 엉뚱한 선이죠.

그래서 "학습"이 뭐냐면 수많은 W, b 조합(=수많은 선) 중에서 데이터에 가장 잘 맞는 W와 b를 찾아내는 것. 그게 이 모델이 "배운다"의 전부입니다.

3. 어느 선이 더 좋은가? → cost(틀린 정도)

선이 좋은지 나쁜지 어떻게 점수 매길까요? 간단합니다. 각 점에서 선까지의 세로 거리를 재는 거예요. 예측 H(x)가 실제 y에서 멀면 나쁜 선, 가까우면 좋은 선.1

한 점의 오차는 H(x) − y 입니다. 그런데 이걸 그냥 다 더하면 문제가 생겨요 — 어떤 점은 +로, 어떤 점은 −로 빗나가서 서로 상쇄돼 버리거든요. 그래서 제곱을 합니다:

( H(x) − y )2

제곱은 두 가지를 한 번에 해결해요: ① 부호를 없애 항상 양수로 만들고, ② 크게 빗나간 점에 더 큰 벌점을 줍니다 (2배 틀리면 벌점은 4배). 이 벌점들을 모든 점에 대해 더하고 개수로 나눠 평균 낸 것이 비용 함수(cost function), 줄여서 cost입니다. (loss라고도 불러요.)1

cost(W, b) = 모든 점의 ( H(x) − y )2 의 평균

핵심은 cost는 W와 b의 함수라는 것. 선(W, b)을 정하면 cost 하나가 딱 나옵니다. cost가 작을수록 좋은 선이에요.

4. 직접 계산해보기 (숫자로 한 번만)

데이터 (1,1) (2,2) (3,3)에 두 선을 대보고 cost를 재봅시다. 곱셈·뺄셈뿐이에요.

선 A — 딱 맞는 선: H(x) = 1·x + 0

x=1 → 예측 1, 실제 1 → 오차 0 → 제곱0
x=2 → 예측 2, 실제 2 → 오차 0 → 제곱0
x=3 → 예측 3, 실제 3 → 오차 0 → 제곱0
cost = (0+0+0) ÷ 30.00

선 B — 엉성한 선: H(x) = 0.5·x + 0

x=1 → 예측 0.5, 실제 1 → 오차 0.5 → 제곱0.25
x=2 → 예측 1.0, 실제 2 → 오차 1.0 → 제곱1.00
x=3 → 예측 1.5, 실제 3 → 오차 1.5 → 제곱2.25
cost = (0.25+1.00+2.25) ÷ 3≈ 1.17

선 A의 cost는 0, 선 B는 1.17. 숫자가 "선 A가 더 좋다"를 자동으로 말해줍니다. 이제 사람의 눈짐작 없이, 기계가 스스로 선의 좋고 나쁨을 판단할 수 있게 된 거예요.

5. 학습의 목표: cost를 최소화하는 W, b 찾기

그래서 선형 회귀의 학습은 딱 한 문장으로 정리됩니다:1

cost(W, b) 를 가장 작게 만드는 W와 b를 찾아라

영어로는 minimize cost(W, b). 이 "가장 낮은 곳을 찾아 내려가는" 방법이 바로 다음 레슨(경사하강법 — 안대 쓰고 언덕 내려가기)의 주제입니다. 오늘은 무엇을 최소화하는지(cost)만 확실히 잡으면 충분해요.
