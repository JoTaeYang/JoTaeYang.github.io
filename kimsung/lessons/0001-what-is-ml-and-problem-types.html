레슨 1 · Phase 0 — 지도 만들기

머신러닝이 뭐고,
문제를 어떻게 "번역"하나

수학은 한 줄도 안 나옵니다. 오늘의 목표는 딱 하나 — 현실의 문제를 보고 "이건 ML로 치면 무슨 종류의 문제다"라고 말할 수 있게 되는 것.

이 레슨을 마치면 할 수 있는 것 업무에서 "이거 머신러닝으로 되나요?" 하는 문제를 만났을 때, 회귀 / 이진 분류 / 다중 분류 중 무엇인지 자신 있게 골라낼 수 있습니다. 이게 실무 ML의 첫 단추입니다.

1. 머신러닝이란: "규칙을 짜지 말고, 배우게 하자"

보통 프로그램은 개발자가 규칙을 전부 적어줍니다. "제목에 '광고'가 있으면 스팸, 링크가 5개 넘으면 스팸…" 이런 걸 명시적 프로그래밍(explicit programming)이라고 해요. 훌륭하지만, 경우의 수가 너무 많은 문제 — 스팸 필터, 자율주행 — 에서는 규칙을 손으로 다 적는 게 불가능합니다.

그래서 1959년 아서 새뮤얼이라는 사람이 이런 생각을 합니다: "규칙을 우리가 짜지 말고, 데이터를 잔뜩 보여줘서 프로그램이 스스로 규칙을 배우게 하면 어떨까?" 이게 바로 머신러닝입니다.1

한 줄 요약 전통 프로그래밍은 규칙 + 데이터 → 답. 머신러닝은 거꾸로 데이터 + 답 → 규칙(모델).

2. 지도 학습 vs 비지도 학습

배우려면 데이터가 필요하죠. 그 데이터에 정답이 붙어 있느냐로 두 갈래가 갈립니다.

  • 지도 학습(Supervised): 정답표가 붙은 데이터로 배움. 고양이 사진마다 "cat" 딱지를 붙여 학습 → 새 사진을 맞힘. 감독관이 정답을 알려주며 가르치는 그림이라 "지도".
  • 비지도 학습(Unsupervised): 정답 없이, 비슷한 것끼리 스스로 묶음. 구글 뉴스가 비슷한 기사끼리 자동 그룹핑하는 것.

이 코스는 실무에서 가장 흔한 지도 학습을 주로 다룹니다. 여기서 두 글자만 기억하세요: 입력은 x(특징/피처, 예: 공부한 시간), 정답은 y(레이블, 예: 시험 점수). 이 (x, y) 짝들의 모음이 트레이닝 데이터셋이고, 이걸로 모델을 학습시킵니다.

3. 오늘의 핵심 기술: 문제를 세 종류로 번역하기

지도 학습 문제는 "정답 y가 어떻게 생겼나"만 보면 셋 중 하나로 딱 나뉩니다. 이 질문 하나만 던지면 됩니다 — "답이 뭐야?"

답이 넓은 범위의 숫자다 ("얼마나?")→ 회귀 (Regression)
답이 둘 중 하나다 ("이거야 저거야?")→ 이진 분류 (Binary)
답이 여럿 중 하나다 ("어느 것?")→ 다중 분류 (Multi-class)

같은 "시험" 문제도 정답을 어떻게 정의하느냐에 따라 종류가 바뀝니다.1 이게 핵심 감각이에요:

업무에 대입해 보기 "다음 달 매출 예상액" → 숫자니까 회귀. "이 고객이 이탈할까 말까" → 둘 중 하나니까 이진 분류. "이 문의는 환불·배송·기타 중 어디로 보낼까" → 여럿 중 하나니까 다중 분류.
