— cost 최소화가 실제 코드가 되는 순간

레슨 4에서 배운 수식 한 줄이, 오늘 진짜 TensorFlow 코드가 됩니다. 역시 한 줄도 실행할 필요 없어요 — 읽고, 출력까지 여기서 확인만.

이 레슨을 마치면 할 수 있는 것 경사하강법 수식 새 W ← 지금 W − α×(기울기)가 코드 3줄(기울기 → 새 W → 대입)로 바뀌는 걸 짚고, 그 3줄을 optimizer 한 줄이 통째로 대신한다는 걸 설명할 수 있습니다.

1. 오늘의 목표 — 그 수식을 코드로

레슨 4에서 "cost 밥그릇 바닥으로 굴러 내려가는" 규칙을 이 한 줄로 적었죠:1

새 W ← 지금 W − α × (기울기)

오늘은 이걸 두 가지 방법으로 코드에 옮깁니다 — ⓐ 수식을 손으로 그대로 옮기기, ⓑ optimizer 한 줄로 맡기기. 그 전에, 레슨 4의 "밥그릇"이 정말 밥그릇인지 코드로 한 번 눈으로 확인하고 갈게요. (데이터는 늘 쓰던 (1,1)(2,2)(3,3), 가설은 단순화해서 H(x)=Wx.)

새로 나오는 Python·TF 조각 (이것만)

list.append(v)목록 끝에 값 v를 하나 추가.
feed_dict={W: 0.5}실행할 때 W 자리에 0.5를 끼워 넣어라.
W.assign(값)Variable W에 새 값을 대입(=)하는 TF 방식. (그냥 W = 값은 안 됨.)

2. 밥그릇 눈으로 확인하기 (cost 시각화)

W를 placeholder로 두고 −3부터 5까지 훑으면서 cost를 재고, 그 값을 찍어보는 코드예요.1

import tensorflow as tf
import matplotlib.pyplot as plt

X = [1, 2, 3]
Y = [1, 2, 3]

W = tf.placeholder(tf.float32)
hypothesis = X * W                              # 단순화: H(x)=Wx
cost = tf.reduce_mean(tf.square(hypothesis - Y))

sess = tf.Session()
sess.run(tf.global_variables_initializer())

W_val, cost_val = [], []
for i in range(-30, 50):                        # W를 -3.0 ~ 5.0 까지
    feed_W = i * 0.1
    curr_cost, curr_W = sess.run([cost, W], feed_dict={W: feed_W})
    W_val.append(curr_W)                        # 나중에 그래프 그리려고 저장
    cost_val.append(curr_cost)

plt.plot(W_val, cost_val)                        # 가로 W, 세로 cost
plt.show()

이 그림을 그리면 — 레슨 4에서 손으로 그렸던 바로 그 밥그릇이 나옵니다. 바닥(cost가 가장 낮은 곳)은 W = 1. 우리가 눈으로 "정답은 1"이라 했던 걸, 코드가 밥그릇 그래프로 그대로 보여주는 거예요.

3. 방법 ⓐ — 수식을 손으로 옮기기

이제 밥그릇 바닥을 자동으로 찾아 내려갑니다. 레슨 4 수식을 딱 세 줄로 옮겨요. 왼쪽 코드와 오른쪽 개념을 나란히 보세요.1

코드 3줄 = 레슨 4 수식 그대로
learning_rate = 0.1
gradient = tf.reduce_mean((W * X - Y) * X)     # ① 지금 지점의 기울기(=미분)를 잰다
descent  = W - learning_rate * gradient         # ② 새 W = 지금 W − α × 기울기
update   = W.assign(descent)                    # ③ 그 새 값을 W에 대입
  • ① gradient — "지금 W에서 밥그릇이 얼마나 기울었나". 레슨 4의 기울기(방향)가 이 줄이에요. (미분 공식을 코드로 적은 것 — 손으로 풀 필요 없이 그냥 옮겨 적음.)
  • ② descent — W − α × 기울기. 레슨 4 수식과 글자까지 똑같죠. α가 learning_rate(보폭).
  • ③ update — 그렇게 구한 새 값을 W에 assign(대입). 이 update를 한 번 실행 = 한 발짝.
실행 — update를 21번 = 21발짝
W = tf.Variable(tf.random_normal([1]))          # 아무 랜덤 값에서 시작
X = tf.placeholder(tf.float32)
Y = tf.placeholder(tf.float32)
hypothesis = X * W
cost = tf.reduce_mean(tf.square(hypothesis - Y))
# ... 위의 gradient / descent / update 3줄 ...

sess.run(tf.global_variables_initializer())
for step in range(21):
    sess.run(update, feed_dict={X: [1,2,3], Y: [1,2,3]})   # ← 한 발짝
    print(step, sess.run(cost, feed_dict={X:[1,2,3], Y:[1,2,3]}), sess.run(W))

4. 예상 출력 — cost는 0으로, W는 1로

(랜덤 시작이라 숫자는 매번 조금 달라도, 1로 수렴하는 방향은 항상 같아요.)

step   cost     W 0   1.68   [1.600]  # 랜덤 시작 1   0.478  [1.320] 2   0.136  [1.171] 3   0.039  [1.091] 5   0.003  [1.026] 10   0.00001 [1.002] 20   0.00000 [1.000]

발짝을 뗄 때마다 cost가 쪼그라들고 W가 1에 착 붙습니다. 레슨 4의 밥그릇 그림에서 공이 바닥으로 굴러가던 그 장면이, 숫자로 눈앞에 찍히는 거예요. 우리가 손으로 경사하강법을 구현했는데 이렇게 잘 됩니다.

5. 방법 ⓑ — optimizer 한 줄에게 맡기기

그런데 ①의 gradient 줄은 미분 공식을 사람이 직접 적은 거예요. 모델이 복잡해지면 이 미분이 끔찍하게 어려워집니다. 그래서 TensorFlow엔 그걸 통째로 대신해주는 게 있어요 — 방금 그 세 줄이 이 한 줄이 됩니다:1

optimizer = tf.train.GradientDescentOptimizer(learning_rate=0.1)
train = optimizer.minimize(cost)                # gradient·descent·assign을 전부 자동으로
레슨 3에서 이미 본 그 줄 레슨 3의 "조각 5 — cost 최소화"가 정확히 이 minimize(cost)였죠. 그때 "마법 버튼"이라 불렀던 것의 속(방법 ⓐ 3줄)을 오늘 열어본 겁니다. 미분을 손으로 안 해도, 텐서플로가 대신 밥그릇 바닥으로 굴려줘요.

6. 어디서 시작해도 1로 (볼록 함수 실증)

레슨 4에서 "밥그릇(볼록)이면 어디서 시작하든 같은 바닥"이라 했죠. 정말 그런지, 시작값을 일부러 엉뚱하게 줘봅니다.1

시작 W = 5.0 (오른쪽 벽 높이)
step   W 0   5.000 1   3.133 2   2.138 5   1.173 20   1.000
시작 W = −3.0 (반대편 벽)
step   W 0  −3.000 1  −1.133 2  −0.138 5   0.827 20   1.000

한쪽은 5.0에서 내려오고 다른 쪽은 −3.0에서 올라오는데, 둘 다 1.0에 도착합니다. 밥그릇 웅덩이가 하나뿐이라 그래요 — 레슨 4의 "볼록이면 시작점 무관"이 코드로 증명된 셈이죠.
