성장 로드맵: 초보 → Kubestronaut

이 트랙의 지도. 각 단계는 자격증 하나로 검증된다.

CNCF 쿠버네티스 자격증 5종(KCNA, KCSA, CKA, CKAD, CKS)을 전부 보유하면 Kubestronaut 타이틀이 부여된다. KCNA/KCSA는 객관식 시험, CKA/CKAD/CKS는 실제 클러스터에서 문제를 푸는 실기 시험(2시간)이다. CKS는 유효한 CKA 보유가 응시 조건이므로 순서가 강제된다.

단계별 지도

단계목표 역량검증 자격증주 실습 환경
0. 기반 컨테이너, 이미지, 쿠버네티스가 풀려는 문제. kubectl 기본기. (진단으로 스킵 가능) kind 로컬 클러스터
1. 생태계 이해 K8s 리소스 모델 전반 + CNCF 생태계(컨테이너 런타임, GitOps, 관측성) 어휘. KCNA (객관식) kind · killercoda
2. 보안 개념 4C 보안 모델, 인증/인가, Pod Security, 공급망 보안 개념. KCSA (객관식) kind · killercoda
3. 앱 개발·배포 Deployment/Service/Ingress/ConfigMap/Secret/Probe/Job 을 맨손으로 작성·디버깅. CKAD (실기) killercoda · killer.sh
4. 클러스터 운영 kubeadm 설치/업그레이드, etcd 백업/복구, 노드/네트워크 장애 해결. CKA (실기) VM + kubeadm · killer.sh
5. 보안 실전 RBAC 최소권한, NetworkPolicy, 런타임 보안, 이미지 스캐닝, API 감사(audit). CKS (실기, CKA 필수) VM + kubeadm · killer.sh
왜 이 순서인가 KCNA·KCSA(객관식)로 지도를 먼저 그리고, CKAD(가장 응용 범위가 좁은 실기)로 손을 만든 뒤, CKA(운영 전반) → CKS(CKA 전제)로 확장한다. 이론→실기, 좁음→넓음, 그리고 응시 조건(CKS←CKA)을 모두 만족하는 유일한 자연스러운 순서다. 순서는 학습자와 합의 후 확정.

단계 통과 기준

  • 각 단계는 해당 자격증 합격으로 종료된다 (실기: killer.sh 모의시험 1회 통과 후 응시).
  • 레슨마다 하나의 skill 을 얻고, 배운 것은 learning record 로 남긴다.
  • 실습은 언제나 일회용 클러스터에서 — 운영 클러스터(mgmt/tkai3)는 이 트랙 범위 밖.
Primary Source 각 자격증의 공식 커리큘럼: github.com/cncf/curriculum — 시험 범위가 바뀌면 이 저장소가 먼저 바뀐다. 단계 시작 전 반드시 확인.
