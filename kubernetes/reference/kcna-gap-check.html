Kubernetes Mastery · Reference · Level 1

KCNA 갭 체크

공식 커리큘럼 도메인 대비 — 커버한 것, 부분인 것, 아직 안 본 것. 시험 준비의 지도.

KCNA 는 90분 객관식(온라인 감독) 시험이다. 정확한 문항 수/비용/정책은 공식 페이지와 커리큘럼 저장소에서 응시 직전에 재확인한다 (도메인 비중도 개정될 수 있다).

도메인별 현황

범례: ✅ 커버 (레슨/실습 증거 있음) · 🔶 부분 (인접 트랙 또는 개념만) · ⬜ 미학습

주제상태근거 / 다음 행동
Pod, 컨테이너, 선언적 모델✅0001, 0002
Deployment / ReplicaSet / 롤링 업데이트✅0002 + 복습 재시험 통과
Service / EndpointSlice / DNS✅0003
probe (liveness/readiness/startup)✅0004 (startupProbe 는 문서 참조로만)
ConfigMap / Secret✅0005
requests/limits, 스케줄링, namespace✅0006
아키텍처 컴포넌트 (API server, etcd, scheduler, kubelet…)✅0007 검증 통과 (제어 루프가 도는 곳 = kube-controller-manager 만 재확인 대상 → 0008 복습에 포함)
기타 워크로드: StatefulSet, DaemonSet, Job, CronJob✅0009 — "언제 Deployment 가 아닌가" (Job 완료 + StatefulSet 안정 신원 실습)
NodePort / LoadBalancer / Ingress✅0010 — ClusterIP(0003) 위에 3단으로 얹기 (NodePort 실습 + LB/Ingress 개념)
PV / PVC / StorageClass✅0011 — PVC=요청/PV=공급/SC=공장, 동적 프로비저닝 + Pod 죽어도 파일 유지 실습
RBAC, ServiceAccount✅0008 실습 통과 — auth can-i 거부→부여→경계를 손으로 확인 (CKA/CKS 재활용)
QoS 클래스 & eviction (심화)✅0020 실습 — Guaranteed/Burstable/BestEffort 판정 + 축출 순서, 0006 OOMKill(137)과 구분
ResourceQuota / LimitRange / Namespace 격리 (심화)✅0022 실습 — namespace 총량 예산 + 기본값 주입, namespace≠네트워크 격리 교정
API 확장: CRD / Operator (심화)✅0021 실습 — CRD 로 API 확장, Operator=CRD+컨트롤러=0007 제어 루프의 day-2 자동화

3. Cloud Native Architecture (~16%)

주제상태근거 / 다음 행동
CNCF 생태계, 프로젝트 성숙도(sandbox/incubating/graduated)✅0014 — Sandbox→Incubating→Graduated 순서·의미 (어휘 수준)
12-factor, 마이크로서비스 vs 모놀리스✅0014 — 12-factor 를 0005(설정)/0002(disposable)/0011(상태 밖) 과 연결

주제상태근거 / 다음 행동

기초 메커니즘 딥다이브 (KCNA 인접 — 이미 배운 것의 진짜 엔진)

시험 밖이라기보다 여러 레슨에 파편으로 흩어져 하나의 그림이 안 되던 밑바닥 메커니즘. 0023~0028 로 한 장씩 굳혀 CKAD/CKA 로 이월.

주제상태근거 / 다음 행동
Pod 생명주기 상태 머신 (phase·컨테이너 상태·restartPolicy)✅0023 — Pending/Running/Succeeded/Failed vs Waiting/Running/Terminated vs Always/OnFailure/Never, 0004·0006·0009 파편 통합
우아한 종료 (SIGTERM→grace→SIGKILL, preStop/postStart)✅0024 — terminationGracePeriodSeconds(기본 30s), 0002 롤링업데이트의 뒷면
Init 컨테이너 & 네이티브 사이드카 (시작 순서)✅0025 — 순서 보장 + 1.28+ 사이드카(restartPolicy:Always initContainer), 0016 메시 밑개념
ownerReferences · GC · finalizer✅0026 — cascade 삭제 메커니즘 + finalizer, 0002 소유 체인의 진짜 엔진
API 그룹·버전·deprecation✅0027 — apps/v1 이 왜 그렇게 생겼나, 그룹/버전/deprecation 정책 리터러시
Static Pod (컨트롤 플레인 부팅)✅0028 — kubelet 이 /etc/kubernetes/manifests 로 mirror pod 생성, 0007 의 다음 층

남은 레슨 계획 (제안)

  1. 아키텍처 + RBAC 검증 퀴즈 — 0007 완료. 아키텍처 ✅, RBAC 은 회상이 흔들려 실습으로 이월.
  2. RBAC 실습 — 0008 완료. auth can-i 거부→부여→경계 통과 → RBAC ✅.
  3. 워크로드 4형제 — 0009 완료. "언제 Deployment 가 아닌가" (Job 완료 + StatefulSet 안정 신원 실습).
  4. 외부 노출 3단 — 0010 완료. NodePort → LoadBalancer → Ingress, ClusterIP 위에 3단으로.
  5. 스토리지 — 0011 완료. PV / PVC / StorageClass, 동적 프로비저닝 + 영속성 실습.
  6. 인터페이스 3형제 + 오케스트레이션 어휘 — 0012 완료. CRI / CNI / CSI, taint/affinity/HPA 개념.
  7. 클라우드 네이티브 어휘 일괄 — 0014 완료. CNCF 성숙도 / 12-factor / 관측성 3기둥 / GitOps / Helm → 저비중 세 도메인 어휘 ✅.
  8. 모의 진단 — 0015 준비 완료. 18문항 전 도메인 → 결과로 최종 갭 확인 후 응시 일정 확정.
전략 비중이 낮은 도메인(관측성 8%, 딜리버리 8%)은 어휘 수준에서 멈춘다 — 과투자는 낭비다. 반대로 Fundamentals(46%)의 ⬜ 항목(워크로드 4형제, 노출, 스토리지)은 실습으로 확실히. 어차피 전부 CKAD/CKA 에서 다시 깊게 만나므로, 여기서의 실습은 이자까지 붙는 투자다.
