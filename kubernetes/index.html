진단에서 시작해 KCNA 전 도메인까지. 각 레슨은 짧고, 하나의 손에 잡히는 win 을 남긴다. 위에서부터 순서대로 보면 된다.

지도: KCNA 갭 체크 (도메인별 커버 현황) · 성장 로드맵 (5-cert Kubestronaut 경로). 실습 클러스터: kind mastery + namespace lab.

#레슨
0002Deployment 는 Pod 를 만들지 않는다

0003Service 는 Pod 를 어떻게 찾아내나
label selector 와 EndpointSlice — 주소가 안 바뀌는 이유.
0004readiness 는 과녁에서 빼고, liveness 는 재시작한다
두 probe 의 차이를 EndpointSlice·RESTARTS 로 확인.
0005설정은 이미지 밖으로: ConfigMap 과 Secret
env 주입 vs volume 주입의 결정적 차이.
0006requests 는 예약, limits 는 단속
Pending 의 원인과 OOMKilled 의 정체.
0008RBAC: 거부를 부여로 뒤집기
auth can-i 거부→부여→경계를 손으로.
0009워크로드 4형제: 언제 Deployment 가 아닌가
StatefulSet·DaemonSet·Job·CronJob — 가정이 깨지는 지점.
0010외부 노출 3단
NodePort → LoadBalancer → Ingress, ClusterIP 위에 3단.
0011스토리지: 요청(PVC), 공급(PV), 공장(StorageClass)
